SQL进阶 Advanced SQL

导言：

     DBMS负责关心query的效率。它完成了对于对算法的封装，对各种操作的的重新排序而产生高效的行为。SQL本质是一种数据库管理系统的标准，并不断更新，标准规定了DBMS需要有的特性。SQL融合了DML+DDL+DCL（Data Control Language）。不同于关系函数基于集合论，SQL实际上基于Bag：无序且允许重复。没有任何现实中的DBMS完全和SQL标准规范的一样，并且每一款DBMS都在尝试把自己的特性融合到“标准”中。这部分内容介绍的是SQL的规范，而非某一种特定的DBMS下的语言。注：这门课程不会专注于如何使用某一门DBMS所提供的语言。







2.1 SQL作为数据定义语言 SQL&Data Definition Language

    SQL DDL提供了对于定义关系Schema，删除rlation，修改关系Schema的指令。具体来说，可以实现一下特性：

  1. Integrity：设置这一关系下的数据必须遵守的限制，超出限制的数据不允许加入；

  2. View Definition：设置视图；

  3. Transaction Control：存在标注一个交易（对于数据的修改）的开始和结束的命令；

  4. Embedded SQL & Dynamic SQL：SQL语句可以被嵌入到其他高级语言中，比如：C，java等等；

  5. Authorization：设置对于视图和关系的访问权限（比如对于数据库用户的某些访问限制）。

    如上所述，SQL可以用来去定义很多内容，比如一个关系的Schema，所保存的数据的类型，限制等等。SQL主要有下面几个数据类型：

  1. char(n)：固定长度的字符串；

  2. varchar(n)：规定最长长度，在这个值之下长度可变的字符串；

  3. int：一个取决于当前计算机规定的长度的int；

  4. smallint：一个小int；

  5. numeric(p,d)：代表某种精度的小数：p规定精度，d规定小数位数，比如numeric(3,1)中，14.5是合法的；

  6. real/double precision：取决于计算机的浮点数（单精度/双精度）

  7. float(n)：n为最小精度，也是浮点数

可以看出来所有取决于计算机本身的类型是不需要（n）的。有时可能会有null出现，比如char(n)实际使用的长度小于n，此时DBMS会把没用到的部分设置为空格，来避免出现null。另外对于varchar，它永远不会有空着的位置，因为长度可变。

    对于Schema的定义：通过 create tabl + tableNam + (attrs varTypes, <contrains>) + ; 的大致格式来创建一个R：

create table department
    (dept name varchar (20),
    building varchar (15),
    budget numeric (12,2),
    primary key (dept name));

这里定义了一个“部门”的R。注：这里使用;来表示定义结束

    SQL支持对于限制的定义，比如：

primary key（A1，Ａ2）设置这个r的Primary key，其中A1，A2是两个属性；

foreign key(A1，A2) references s 设置引用于s，另一个Table，的这些（A1，A2）属性。如果r中使用了这些属性但是赋值了s中没有出现的值，那么说 “限制被破坏了”。例如可以：“foreign key (dept name) references department；

not null使得一个属性的值不可是NULL。使用：name varchar (20) not null，加载属性类型后面。

    SQL支持对r的删除和对r中的属性的删除和添加：

drop table r：这个命令不管会删除该关系中的所有数据，也会把目标Schema删除；

delete from r：保留关系，只删除数据；

alter table r add A D：在关系r中添加一个属性A，其数据类型为D。在添加之后，这个属性对应的数据将都是NULL（初始值）；

alter table r drop A：删除属性A，大多DBMS不支持。







2.2 SQL查询基础结构 Basic Structure of SQL Queries

    对于查询的使用实际上是SQL的DML的体现，这里可以看到对于关系函数的真正的使用。

    最基础的SQL数据库查询由三个语句（从句Clause）组成：select, from, where。在一个查询中，from后面的内容会被看作是输入的r，对于输入的数据的操作在另外两个语句中明确。使用英文中的从句关系可以按阅读顺序知道一整个查询在干什么。

2.2.1 单关系查询 Queries on a Single Relation

    单关系查询即输入的关系中只有一个关系，比如：

select name 
from instructor;

从名为instructor的关系中找出了所有name属性对应值，输出为一列数据，第一行为 name，下面是每个instructor的name对应的值。可以使用select distinct name 去重，不加distinct实际上默认使用了关键字 all，即select all name 。

    与此同时，在select语句中也可以对所选择的属性做出一定计算：

select name, salary *1.1 
from instructor;

在输出的r中，会有两列，第一列为name，第二列为salary，由于`*1.1``的存在，salary会全部乘以1.1。这计算和修改只在输出的r中呈现，不会对原r有任何影响。

     where语句实际上就是对于谓词的描述：

select name 
from instructor 
where dept name = 'Comp. Sci.' and salary > 70000;

表示：“Find the names of all instructors in the Computer Science department who have salary greater than $70,000”。

    SQL中所支持的逻辑连接词：and, or, not；比较运算符：<, <=, >=, =, <>。

注：select这里并不完全是关系代数中的选择，同时也是投影。

2.2.2 多关系查询 Queries on a Multiple Relations

    与单关系类似，但不同的关系之间使用逗号分隔，且两个关系中同名的属性需要使用 r.attr 标注清楚使用的是谁的：

select name, instructor.dept name, building 
from instructor, department 
where instructor.dept name= department.dept name;

意为： “Retrieve the names of all instructors, along with their department names and department building name.”

一个简单的查询结构是这样的：

select A1, A2, … , An 
from r1, r2, … , rm 
where P ;

每一个A代表一个属性，每一个r代表一个关系，P是谓词。当where语句不存在时，默认P为True。

    本质上 from 语句是对每个输入的关系做之前介绍的笛卡尔积，而where决定的谓词用来限制笛卡尔积产生的结果。所以可以把一个输出的过程看作：

  1. 通过from产生关系的笛卡尔积；

  2. 通过where决定的谓词少选上面的结果；

  3. 使用select选择筛选后的目标属性并输出。

这或许并不是SQL底层实现输出的顺序，因为存在优化。

补充：在select中，可以出现类似 select instructor.* 这样的写法，来选择“r的所有属性”：这样输出的关系中就全是谓词筛选后的原来instrustor中的数据；除了理解为对输入的关系的笛卡尔积的操作，from和where也可以看作是多个关系的循环体：

for each tuple t1 in relation r1
    for each tuple t2 in relation r2
        … 
            for each tuple tm in relation rm
                Concatenate t1, t2, … , tm into a single tuple t 
                Add t into the result relation






2.3 基础操作 Basic Operations

     除了上述的三个基本操作，SQL还有其他大量的操作。


    重命名：使用 as 关键字，可以为select输出的属性重命名，也可以为输入的关系重命名：

select name as instructor_name, course_id 
from instructor as T, teaches
where T.ID= teaches.ID;

这里重命名了name 为 instructor_name，instructor 为 T。后者可以被使用用于where中以及select中。对于输入的关系的重命名有多种叫法： correlation name, table alias, correlation variable, tuple bariable。值得注意的是对于输出属性的重命名会在输出的关系中以重命名后的名字出现，而对于输入的关系的重命名不会对原数据有任何影响：它只作为别名存在。


    字符串操作（String）：SQL使用单引号 括一个字符串，而单引号内部的引号是用双引号，比如 “It’s right” 在 SQL中表示为 ‘It"s right’。按SQL的标准来讲，字符床之间的比较应当是大小写敏感的，但是不同的DBMS有不同的规则，并且这一规则也可能可以通过一些行为修改。SQL还允许很多字符串的函数，比如连接，提取子串，找出长度，转换为大写/消协，删除最后的空格等等，这些函数可以出现在任意从句中。

    同时也存在模式匹配，需要借助like关键字使用：

  1. %: 可以代表任何字母/子串
  2. _: 可以代表任何字母

模式匹配是大小写敏感的。例子：

‘Intro%’ matches any string beginning with “Intro”.

‘%Comp%’ matches any string containing “Comp” as a substring, for example, ‘Intro. to Computer Science’, and ‘Computational Biology’.

‘_ _ _’ matches any string of exactly three characters.

‘_ _ _%’ matches any string of at least three characters.

select dept_name 
from department 
where building like '%Watson%';

意为：“Find the names of all departments whose building name includes the substring ‘Watson’”，即前后无论是什么都可以。

    转义字符“\：用来把有特殊意义的符号变成符号本身。比如：

‘ab∖%cd%’ escape ‘∖’ matches all strings beginning with “ab%cd”.

‘ab∖∖cd%’ escape ‘∖’ matches all strings beginning with “ab∖cd”.

    SQL还允许找出“不匹配”，通过not like关键字实现，与like的使用一样，但是结果相反。还有一些SQL的DBMS允许similar to，它实现了更高阶的正则匹配。


    排序：使用关键字order by实现：

select name 
from instructor 
where dept name = 'Physics' 
order by name;

这样输出的结果中就是以name的字母顺序排序了。这里的排序默认升序，但是可以手动设置：

select * 
from instructor 
order by salary desc, name asc;

这里在属性后面标注了排序的规则，desc为降序，asc为升序。同时设置了两个属性的排序，他们的优先级不一样：如果两个tuple的salary一样，那么就对比name。


    范围谓词：如之前所说，谓词中可以出现比较符号，但是如果想要表达对于一个区间的谓词时，有另一种写法。这是之前所知的选法：

select name 
from instructor 
where salary <= 100000 and salary >= 90000;

表达为选出特定薪资范围内的教师，它也可以写成：

select name 
from instructor 
where salary between 90000 and 100000;

即使用between and关键词。

    除此以外，SQL还允许在谓词中使用 row constructor 来对多个属性进行缩写比较。比如：

select name, course id 
from instructor, teaches 
where instructor.ID= teaches.ID and dept name = 'Biology';

希望同时比较两个属性，并且比较符号相同，那么就可以简写为：

select name, course id 
from instructor, teaches 
where (instructor.ID, dept_name) = (teaches.ID, 'Biology');

    输出关系： 把一个select产生的结果保存到注空中作为一个关系存在：使用 into 关键字：

select distinct cid into CourseIds
from teaches;

这样不需要提前定义关系CourseIds，因为DBMS会自定从select产生的结果中知道新的关系的属性和类型等等信息自动创建。不同的DBMS这里的语法会不一样。

    同时也可以把一个select产生的结果插入到一个现有的关系中，使用insert into实现。需要注意，select产生的结果的属性，类型等等必须和目标关系完全一致。当出现不一致的时候，不同的DBMS会有不同的反应。

    限制输出：有时候可能会希望只select前n个Tuples，使用limit实现：

select name, course id 
from instructor, teaches 
where instructor.ID = teaches.ID
limit 20 offset 10;

这样的操作可以跳过（offset）前十个Tuples，显示下20个Tuples。浏览器的分页就是这样的







2.4 集合操作 Set Operations

    回忆关系代数，SQL也存在类似的集合操作: union, intersect, except。下面通过这两个查询为例：

select course id 
from section 
where semester = 'Fall' and year= 2017;

select course id 
from section
where semester = 'Spring' and year= 2018;

    并集：使用关键字union：

(select course_id
from section
where semester = 'Fall' and year= 2017) 
union
(select course_id
from section
where semester = 'Spring' and year= 2018);

不同于select语句，union会自动去重。由于输出的结果中属性一致，所以只是前一个输出结果后面接上后一个的输出结果（去重）：在这个例子中，输出的是2017年秋季和2018年春季所有的课程。可以手动开启允许重复：把union替换成union all 即可。

    交集：使用关键字intersct：

(select course_id
from section
where semester = 'Fall' and year= 2017) 
intersect 
(select course_id
from section
where semester = 'Spring' and year= 2018);

这里选出的是在两个学期中都出现的课程，也自动去重。同样地，使用 intersect all 允许重复。

    差集：使用关键字except：

(select course_id
from section
where semester = 'Fall' and year= 2017) 
except 
(select course_id
from section
where semester = 'Spring' and year= 2018);

这里的输出结果为在第一个学期出现，但是不在第二个学期出现的所有课程，与交集的输出完全相反。使用except all允许重复。







2.5 Null值 Null Values

    Null值在关系操作中会带来一些问题，比如算数操作，比较操作，集合操作。

    算数：任何数与Null的计算结果都是Null。

    比较：任何数与Null比较的结果都是Unknown。Unknown是一个逻辑值，是区别于true，false之外的第三个。不同的逻辑连接符对于unknown的处理不同：

  1. and：True and Unknown = Unknown; False and Unknown = False; Unknown and Unknown = Unknown;

  2. or：True or Unknown = True; False or Unknown = Unknown; Unknown or Unknown = Unknown;

  3. not：not Unknown = Unknown.

在谓词中，结果为Unknown的运算不会进入输出。

    在SQL存在关键字 is/is not null, is/is not unknown来对这两个值进行谓词中的判断：

select name 
from instructor 
where salary is null;

select name 
from instructor 
where salary > 10000 is unknown;

注：unknown是一个逻辑值，所以在谓词中加在bool返回值之后，而null是一个数值，加在属性后面。在使用 select distinct时，如果两个相同属性的值都是null，那么可以判定为重复，也就是说null==null返回True：这是一个例外情况。







2.6 聚合函数 Aggregate Functions

    聚合函数（Aggregate Functions） 使用一个或多个集合的制作为输入，然后输出一个值。SQL中内置5个标准聚合函数：avg, min, max, sum ,count。


2.6.1 基础聚合 Basic Aggregation

    比如期望寻找某个部门的教师平均薪资，可以用过aavg来对salary属性进行计算：

select avg (salary) as avg_salary 
from instructor 
where dept_name = 'Comp. Sci.';

avg(…)的输出结果为一个全新的属性（同时也是一个r），所以可以重命名，在这里这个属性只会有一个tuple。这里体现出了select保留重复的重要性：如果删除了某个教师，会导致输出不了想要的结果。

    聚合函数还可以和其他关键字搭配使用：

select count (distinct ID) 
from teaches 
where semester = 'Spring' and year = 2018;

select count (*) 
from course;

第一个操作计算出了目标学期中有几节不同的课（通过distinct去重），第二个操作所有计算课程数量。可以看出来，对于关键字的理解不应当附着与select等语句：它们本质上也是对属性做出的行为。

select *
from instructor

2.6.3 Having 语句/从句

     having是一个新的从句，可以说它是对于group来讲的“where”，也就是对于每组数据的谓词。

select dept_name, avg (salary) as avg_salary 
from instructor 
group by dept_name 
having avg (salary) > 42000;

这里会把被谓词筛去的分组在输出中删忽略。having和where可以同时出现，不矛盾。

注：where中不可以使用聚合函数结果的重命名（比如不能where avg_salary>42000`），但是having可以; where的判断先于having。

    基于之前的搜索基本结构处理顺序，考虑这两个新的从句后，大致是这样的：

  1. 通过 from 得到输入关系/关系的笛卡尔积；

  2. 如果有 where 存在，那么把where中的谓词施加于from得到的关系中；

  3. 谓词为真的Tuples会被group by处理并分组，如果gourp by不存在，那么相当于把所有数据放到了一个组中（而并非“不分组”）；

  4. 如果 having 存在，对每个组进行筛选忽略；

  5. 使用select生成最终的结果，若聚合函数存在，则以每个group为Tuple基于对应的值。

注：在所有聚合函数中，除了count(*)，对于目标关系中的null值的处理都是直接忽略，比如：

select sum (salary) 
from instructor;

中如果有的教师的salary的值时null，就不会统计他的salary。同时，由于null被忽略，可能导致输入是空的（即全部是null），此时count返回0，其他聚合函数返回null。







2.7 嵌套子查询 Nested Subqueries

    子查询就是嵌套在某个语句中的查询，其返回值作为该外部语句的输入。


2.7.1 Where中的子查询 Nested in Where

    SQL允许测试tuples是否存在于某个关系中，通过连接词 in/not in 实现。之前的例子中有使用交集解决 找到所有在两个学期中都出现过的课程，而现在也可以使用子查询+in解决：

select distinct course_id 
from section 
where semester = 'Fall' and year= 2017 and
    course_id in (select course_id
                  from section 
                  where semester = 'Spring' and year= 2018);

这里把2018春季课程作为输入，检查2017秋季课程是否在前者之中，若存在，就把该Tuple放入输出的关系中。由于逻辑上这还应当是一种交集，所以需要在select中使用distinct来去重（无论是否2017秋季课程中本身是否有重复，因为其目的为实现“交集”）。

    与此对应的是not in关键字，它实际上就是 except，差集，的另一种实现：

select distinct_name 
from instructor 
where name not in ('Mozart', 'Einstein');

所以被检查的集合没必要一定是一个Table，其本身是一个原生的集合也可以。

    有的DBMS支持这样的写法：

select count (distinct ID) 
from takes 
where (course id, sec id, semester, year) in (select course id, sec id, semester, year
from teaches 
where teaches.ID= '10101');

一次检查多个属性，表达“find the total number of (distinct) students who have taken course sections taught by the instructor with ID 110011”。类似于使用双重循环，把当前Table中的一个Tuple的这些属性和另一个Table中的对应属性进行遍历对比。

     some/all：除了in，some和all也可以作为关键字通过一个输入的集合筛选另一个集合。这一行为可以通过逻辑连接词判断多个条件实现：

select distinct T.name 
from instructor as T , instructor as S 
where T.salary > S.salary and S.dept_name = 'Biology';

意为：找到所有比生物部门任意教师的工资高的教师。这里把instructor命名为了两个不同的变量，形成了一个笛卡尔积，这也使得下面的T.salary > S.salary可以遍历所有教师与其他所有教师。同时SQL中允许：

select name 
from instructor 
where salary > some (select salary
                     from instructor 
                     where dept_name = 'Biology');

它首先找到了所有所有生物部门的教师并得到了他们的工资，然后之外部查询直接检查谓词是否成立。SQL中存在> some, < some, <= some, >= some, = some,<> some，其中= some, <> some 等同于 in, not in。all的使用完全一致，只是所规范的范围更加极端，并且<> all = not in但是= all不是in。

    （not）exists用来检查某一个子查询/集合是否为空集：

select course_id 
from section as S 
where semester = 'Fall' and year= 2017 and
        exists (select *
                from section as T 
                where semester = 'Spring' and year= 2018 and S.course_id= T.cours_id);

这是对之前的两个学期共有的课程的另一种写法，解释为某节课在2017秋季提供，并且其课号在2018年春季也出现。如果其中的子查询返回空集，exists返回False，导致某课程不会被纳入最终输出。not exists即相反。

    观察上面的例子，很重要的一个特征是它在子查询中使用了外部查询定义的别名。在SQL中规定，一个查询只能使用它自己定义的别名和其外部查询定义的别名，前者叫做局部定义，后者叫做全局定义（这里的全局和局部是相对来说的，如果这个子查询也有子查询，对于后者，前者的局部定义是全局定义）。

    (not) unque检查其内部的子查询是否有重复Tuples，没有则返回True，若返回为空，也是True：

select T.course_id 
from course as T 
where unique (select R.course_id
              from section as R 
              where T .course id= R.course id and
                    R.year = 2017);

意为： “Find all courses that were offered at most once in 2017”。这里通过循环体的方式更好理解。


2.7.2 From中的子查询 Subqueries in the From Clause

     假如想 “Find the average instructors’ salaries of those departments where the average salary is greater than $42,000”，可以通过之前的having语句搭配group by实现，而这里通过给from使用子查询实现：

select dept_name, avg_salary 
from (select dept_name, avg (salary) as avg_salary
      from instructor 
      group by dept_name)
where avg salary > 42000;

这里把子查询的输出结果作为外部查询的输入关系。这里不需要再使用having，因为子查询把部门名作为了分组，而对于外部查询来说这一分组就是输入的关系的Tuple，而where对这些tuple的筛选代替了having。与普通情况类似，外部查询可以使用子查询中select给出的属性（毕竟实际上就是子查询输出的关系）。

    通过这样产生的输入关系也一样可以起别名：

select dept_name, avg_salary 
from (select dept_name, avg (salary) 
      from instructor 
      group by dept_name)
      as dept_avg (dept_name, avg_salary)
where avg salary > 42000;

这里给子查询输出的关系命名，同时也给属性命名。注：as在外部查询中，不是子查询的一部分。不同的DBMS这里会有不同的特殊规定。

    当from中有多个输入的关系时，若一个输入是一个子查询，它无法使用其他输入关系的别名， SQL提供lateral关键字解决：

select name, salary, avg _alary 
from instructor I1, lateral (select avg(salary)as avg_salary
                             from instructor I2 
                             where I2.dept_name= I1.dept_name);

2.7.3 Select中的子查询 Nested Queries in select

    select中也可以嵌入子查询：假设想要查询某一个课程中的学生名字，可以使用where子查询：

select name 
from student
where sid in(
    select sid 
    from enrolled
    where cid='15-445'
)

这里实际上就是首先产出了子查询的搜索结果，然后让外部查询中的每个tuple的sid元素查看它是否也在子查询的关系中。这一查询也可以改写为selec子查询：

select (select S.name 
        from student as S
        where S.sid = E.sid) as sname
from enrolled as E
where cid='15-445'

这样修改了查询的过程：首先找到所有enrolled中Tuple的cid=‘15-445’的，然后用它们再和student关系中的sid相对比。

    两种写法产生的结果完全一样，但是由于操作顺序改了，性能可能不一样。







2.8 窗函数 Window function

    窗函数类似聚合函数，但是tuples不会被压缩成为1个Tuple。基本结构为：

select ... Func_Name(args) over(...)
from r

这里的Func_Name是聚合函数和一些特殊的窗函数，over是一个从句，限定切分数据的方式，类似group by。 特殊的窗函数：

row_number(): 现在的行数；

rank(): 当前行的位置。

row_number()使用：

select *, row_number() over() as row_number
from enrolled;

结果会把r原r完整输出，但是每一个tuple的后面都有当前这个tuple所在的行的函数，从1～n。由于是对所有tuple统一使用的，over()不用填参数。但如果希望把数据分组，则需要：

select cid,dis, 
    row_number() over(partition by cid) as row_number
from enrolled;

在这个输出中，有相同cid的数据将会被放到相邻的位置，然后row_number表达的是当前tuple在其所在的group中的位置：

    over中还可以规定其他内容，比如一个group内部的排序：

select * from(
    select *,
        rank() over(partition by cid
                    order by grade asc)
        as rank
     from enrolled) as ranking
where ranking.rank=1;

在这个例子中，子查询使用了over，导致输出中的tuples以cid分组，每个组中以grade升序排序，同时使用窗函数rank以及 as rank，给每个tuple一个rank（rank类似row_number，子不过前者给出的是一个tuple在排序中的“rank”，且遇到重复的tuple会给予相同的数字，如果没有order by，rank函数无意义）。外部查询使用谓词找到所有rank为1的tuples。

    窗函数/聚合函数+over的组合和聚合函数+group by的组合有些类似，但是后者会在输出关系中看不到原关系，而前者不会。比如：

select max(grade)
from enrolled 
group by cid

enrolled中保存着所有学生和选课。这个代码会产生一列数据，每一行代表所有相同的cid为一组中分数最高的学生的分数。输出结果类似

2.9 公共表表达式 Common table expressions

    公共表表达式CTE实际上是对with从句的使用： with从句可以创建一个临时关系：

with max_budget (value) as
    (select max(budget)
     from department) 
select budget 
from department, max_budget 
where department.budget = max_budget.value;

可以看到，完全可以使用from位置的子查询来达到相同的目的。但是由于子查询不易读，with从句提高阅读性。with去要搭配as使用，需要同时定义临时关系的名字和属性名（对应于as后面子查询的select）。可以有多个as创建多个临时关系：

with dept_total (dept_name, value) as
    (select dept_name, sum(salary)
    from instructor
    group by dept_name), 
    dept_total_avg(value) as
        (select avg(value)
         from dept_total) 
select dept_name 
from dept_total, dept_total_avg 
where dept_total.value > dept_total avg.value;

注：在with/as创建一个临时关系后，只有后面的代码可以使用它，包括后面的as；不同的临时关系使用逗号分隔。

    除了易读性，CTE同时也提供了递归：

with recursive cteSource (counter) as (
    (select 1)
    union all
    (select counter + 1 
     from cteSource
     where counter<10)
)
select * from cte Source

这里的with语句中使用了它自己定义的关系以及属性，并且通过union把递归的结果union到一个关系中。counter<10作为Base case存在。在递归中union是必要的，因为需要把不同的关系放在一起。

    SQL中允许子查询以及CTE，因为在理想的情况下最好把所需的查询写在一个查询里，这样方便DBMS去优化。
