Chap6: 贪心算法

目录



导言：

    本章着重于贪心算法/思想的介绍和正确性证明。贪心算法不同于DP，没有具体的套路，这导致贪使用贪心思想解决问题可能导致大部分“贪心“都是没有正确性的，而有正确性的贪心又很难证明。







6.1 区间问题

    区间问题是贪心算法的一大类。虽然贪心没什么套路，但是可以尝试全照左右端点排序来实验。



6.1.1 区间选点

  需求：

给定 $N$ 个闭区间 $[a_i,b_i]$，请你在数轴上选择尽量少的点，使得每个区间内至少包含一个选出的点。输出选择的点的最小数量。位于区间端点上的点也算作区间内。

输入格式:

第一行包含整数 $N$，表示区间数。

接下来 $N$ 行，每行包含两个整数 $a_i,b_i$，表示一个区间的两个端点。

输出格式： 输出一个整数，表示所需的点的最小数量。

#define N 100010

//使用一个结构体储存一个区间，保存左右两个端点
struct Range{
    int l,r;
    //重载运算符，供sort使用
    bool operator<(const Range &w) const{
        return r<w.r;
    }
    //结构体数组
}range[N];

int main()
{
    int n,m;
    scanf("%d",&n);
    int l,r;
    //赋值
    for (int i = 0; i < n; i ++ ){
        scanf("%d%d", &l, &r);
        range[i]={l,r};
    }
    //以右端点排序
    sort(range,range+n);
    //ed代表区间的右端点，初始化为最小值
    int res=0,ed=-2e9;
    
    for (int i = 0; i < n; i ++ ){
        //如果发现有某个区间的左端点比当前使用的右端点大
        //说明这个右端点没有办法覆盖这个区间
        if(range[i].l>ed){
            res++;
            //更新右端点
            ed=range[i].r;
        }
    }
    printf("%d\n",res);
    return 0;
}

    区间选点的主要思想就是：选择某一区间的右端点，比较这个右端点的值和其他区间的左端点。假如前者比后者大，说明后者所在的区间被前者所在的区间覆盖，反之则不覆盖。

  正确性证明：     设正确答案为$Ans$，设贪心得到的结果为$cnt$。我们想求得$Ans=cnt$，可以通过$Ans\leq cnt$和$Ans \geq cnt$ 证得。

    $Ans\leq cnt$: 由于这个算法在最糟的情况下也会为每个区间分配一个点，即无论如何该算法得到的结果一定可以把每个区间覆盖，即使点的数量不是最小。所以我们可以得到：$Ans\leq cnt$。

    $Ans \geq cnt$：已知该算法看到当某个区间被另一个区间的右端点的时候，不会增加res，而只有在两个区间完全不重合的时候才会增加。假设被覆盖的区间从图中删除，那么最后我们可以得到这样一个图像：

区间之间彼此不重合，为了让所有区间都被点覆盖，那么需要这些区间个数个点来覆盖它们（只能自己覆盖自己了）。该算法会取到这些区间的右端点来覆盖所有区间，而任何其他可以覆盖到所有区间的结果都会大于等于该算法得到的结果，所以$Ans \geq cnt$。



6.1.2 最大不相交区间数量

  需求：

  给定 N 个闭区间 $[a_i,b_i]$，请你在数轴上选择若干区间，使得选中的区间之间互不相交（包括端点）。输出可选取区间的最大数量。

输入格式：

  第一行包含整数 N，表示区间数。接下来 N 行，每行包含两个整数 $a_i,b_i$ ，表示一个区间的两个端点。

输出格式：

  输出一个整数，表示可选取区间的最大数量。

    本体提议可以理解为一天的活动安排：在有限的时间中经可能多的规划活动。这道题的代码和上一道题完全一样。

  正确性证明：

    设正确答案为$Ans$ 为最大值，设贪心得到的结果为$cnt$。我们想求得$Ans=cnt$，可以通过$Ans\leq cnt$和$Ans \geq cnt$ 证得。

    $Ans \geq cnt$：$cnt$ 得到的结果一定是一个合法的结论因为如上题中，每个点最终实际上依附在了 $cnt$个不相交的区间上，所以是一个可行的方案。而$Ans$是其中最大的可行方案，所以$Ans \geq cnt$。

    $Ans\leq cnt$: 用反证法，假设$Ans > cnt$，也就是说上题图中某两个区间内存在一个小区间被$Ans$ 考虑但是没有被 $cnt$ 考虑。此时发现矛盾：假如这个小区间存在，它会被独立地考虑为一个区间并导致res++。



6.1.3 区间分组

  需求：

给定 N 个闭区间 $[a_i,b_i]$，请你将这些区间分成若干组，使得每组内部的区间两两之间（包括端点）没有交集，并使得组数尽可能小。输出最小组数。

输入格式：

  第一行包含整数 N，表示区间数。接下来 N 行，每行包含两个整数 $a_i,b_i$，表示一个区间的两个端点。

输出格式：

  输出一个整数，表示最小组数。

    思路：

  1. 把所有区间按左端点排序；
  2. 从前往后处理每个区间: 判读是否能把它放到现存的某个区间的集合中，即 这个区间的左端点大于这个集合所有区间中最靠右的右端点（最后一个区间的右端点一定是最大的右端点因为一个集合内的所有区间不重合） 1.如果不存在这样的集合，就开另一个新的集合并放入 2.如果存在就放入并且更新最大右端点

    正确性证明：仍然通过$Ans\leq cnt$和$Ans \geq cnt$ 证明。

    $Ans\leq cnt$：这一思路的结果必然大于等于正确结果因为它保证了它得到的结果是合法的：每一个组内的区间都没有交集。

    $Ans\geq cnt$：根据反正发假设$Ans < cnt$。那么可知通过该算法得到的所有区间构成的这些集合中，有至少一个集合可以被分配到其他的集合中。这里观察到冲突：假如陈述为真，那么意味着该算法忽略了某种情况，把没有交集的区间分到了不同的集合，但这种情况不存在（即不存在超出上述两个情况的第三种）。

6.1.3 区间覆盖

     假设一个人在时间区间$[S,F)$ 看电视，已知$n$ 个电视剧，每个占用时间区间$[s_i,f_i)$，完全覆盖了这一时间区间，求在完全占满时间区间$[S,F)$的同时，切换最少频道的数量。

输入格式：      第一行包含两个整数 $s$ 和 $t$，表示给定线段区间的两个端点。第二行包含整数 $N$，表示给定区间数。接下来 $N$ 行，每行包含两个整数 $a_i,b_i$，表示一个区间的两个端点。

#define N 1000010

//range[N]结构体数组不变

int main()
{
    int s=0,t,n,res=0;
    bool success=0;
    cin>>s>>t>>n;
    for (int i = 0; i < n; i ++ ) {
        scanf("%d%d", &range[i].l, &range[i].r);
    }
    sort(range,range+n);
    
    for (int i = 0; i < n; i ++ ){
        int j=i,r=-2e9;
        //当遇到第一个与当前区间不重合的区间则跳出while循环
        while(j<n&&range[j].l<=s){
            //找到当前range[i]所覆盖到的区间中右端点最靠右的
            r=max(r,range[j].r);
            j++;
        }
        //这里说明第一个区间没有从0开始，则无法满足要求
        if(r<s){
            res=-1;
            break;
        }
        res++;
        //超出限制证明整个区间已经被覆盖
        if(r>=t){
            success=1;
            break;
        }
        //把选出的最靠右的端点当作下一轮的左端点
        s=r;
        //上面导致while循环跳出的区间还会被再考虑一次
        i=j-1;
    }
    if(!success) res=-1;
    cout << res;
    return 0;
}

实质上每个区间最终只被考虑了一次，所以时间复杂度为$\mathcal{O}(n)$。
