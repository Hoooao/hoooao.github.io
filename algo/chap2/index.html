    本章主要介绍一些常用的数据结构的实现，包括单链表，双链表，栈，队列，单调栈，单调队列，KMP，Trie，并查集，堆，哈希表等内容。注：本章着重于数据结构模拟，即用数组模拟上述数据结构：数组模拟的数据结构比STL快。







2.1 链表

    链表的实现有两种，第一种是通过结构体保存当前节点的数据和指向下一个/上一节点的指针，第二种是通过数组模拟链表，即用多个数组分别代表第 idx 个节点的数据，上下节点（静态链表）。



2.1.1 单链表

    单链表只保存下一个节点的“指针”。模拟方式：通过一个数组记录节点的值（e），一个数组记录下一个节点的指针(ne)。

如上图所示，它代表着一个由数组模拟的链表。Head可以理解为一个角标，它的值是第一个节点的idx，形式上是“指向”头节点。头节点的值为10，idx为0，它的下一个节点的idx为1。第二个节点则指向idx为5的元素，后者是尾节点。连续的节点的idx不一定连续，因为可能存在删除的操作。在数组模拟中，删除的节点实际上是直接被其上一个节点的ne跳过了，但实际上仍然存在于这个数组之中。虽然会造成空间的浪费，但是当节点数据特别多的时候，数组模拟创建速度比创建struct快得多，且后者能做的事情前者全能做。

需求：

  1. 在头部添加元素：给e[idx]赋值，让ne[idx]指向head所指向的节点（原头节点）并且让head等于idx，让它成为头节点。
  2. 在k节点的后面添加节点。这里的k是节点idx的值。类似添加头节点，只不过把head换成了某个节点
  3. 删除节点k后面的节点：实际上就是让k的ne[k]跳过下一个节点，即等于下一个节点的ne。
#include <iostream>
using namespace std;
#define N 100010
int m,head,idx;
int e[N],ne[N];

//初始化链表，用-1代表null，即尾部
void init(){
    head=-1;
    //idx从0开始分配，代表着共有几个节点被创建
    idx=0;
}

void remove (int k){
    //跳过下一个点
    ne[k]=ne[ne[k]];
}

void addH(int x){
    //这里可以合并成一行写，是用逗号分隔
    e[idx]=x, ne[idx]=head, head=idx++;
}

void add(int k, int x){
    e[idx]=x, ne[idx]=ne[k], ne[k]=idx++;
}

int main()
{
    char a;
    int b,c;
    init();

    //a是输入的指令
    cin>>a;
    if(a=='H'){
        cin>>b;
           addH(b);
    }
    if(a=='D'){
        cin>>b;
        if(!b) head=ne[head];
        else remove(b-1);
    }
    if(a=='I'){
        cin>>b>>c;
        //-1是因为idx从0开始
        add(b-1,c);
    }
    //遍历需要通过寻找下一个节点完成而不是i+1，因为idx不连续
for (int i = head; i != -1; i = ne[i]) cout << e[i] << ' ';
return 0;
}


2.1.2 双链表

    双链表即有指向左右节点的指针的链表，同样可以通过数组模拟。

//m：操作次数
int m,idx;
//这里的l和r分别代表一个节点做节点和右节点的idx
int e[N],l[N],r[N];

//在idx=a后面添加节点
void add(int a, int x){
    //赋值
    e[idx]=x;
    //搭建x
    l[idx]=a,r[idx]=r[a];
    //重构r[a]和a
    l[r[a]]=idx,r[a]=idx++;
}

void remove(int a){
    //把a跳过去
    l[r[a]]=l[a];
    r[l[a]]=r[a];
}

int main()
{
    //初始化
    //创建头节点和尾节点
    //这样相当于已经搭建了1个链表
    //解释为头节点的右边的节点是1，1是右端点
    l[1]=0, r[0]=1;
    idx=2;
    cin>>m;
    string op;
    //x：操作的值；b：操作的idx
    int x,b;
    while(m--){
        cin>>op;
        //首插
        if(op=="L"){
            cin>>x;
            add(0,x);
        }
        //尾插
        if(op=="R"){
            cin>>x;
            //即给右端点左边的节点后面插入节点
            add(l[1],x);
        }
        //给指定元素左边插入
        //b+1实际上是b+2-1，+2是因为最开始已经有两个节点了
        //-1是因为第n个插入的节点实际上idx是n-1
        if(op=="IL"){
            cin>>b>>x;
            add(l[b+1],x);
        }
        //给指定元素右边插入
        if(op=="IR"){
            cin>>b>>x;
            add(b+1,x);
        }
        //删除指定的节点
        if(op=="D"){
            cin>>b;
            remove(b+1);
        }
    }
    for (int i = r[0]; i != 1; i = r[i]) cout << e[i] << ' ';
}






2.2 栈和队列

    同样，本节介绍通过数组模拟栈和队列以及单调栈/单调队列。栈和队列的实现更加简单。



栈：LIFO：last in first out

//m：操作次数
int m;
//使用stk代表stack，tt代表栈顶
//默认栈顶从1开始
int stk[N], tt;

int main()
{
    cin >> m;
    while (m -- )
    {
        string op;
        int x;

        cin >> op;
        if (op == "push")
        {
            cin >> x;
            stk[ ++ tt] = x;
        }
        else if (op == "pop") tt -- ;
        else if (op == "empty") cout << (tt ? "NO" : "YES") << endl;
        //返回栈顶
        else cout << stk[tt] << endl;
    }
    return 0;
}

单调栈：本质上就是一个栈，维护栈内元素保持单调，使用单调性优化问题。

例题：给定一个长度为 $N$ 的整数数列，输出每个数左边第一个比它小的数，如果不存在则输出 −1。

输入格式:

  第一行包含整数 N，表示数列长度。

  第二行包含 N 个整数，表示整数数列。

输出格式:

  共一行，包含 N 个整数，其中第 i 个数表示第 i 个数的左边第一个比它小的数，如果不存在则输出 −1。

数据范围:

  $1≤N≤10^5$   $1≤$ 数列中元素$≤10^9$

#define N 100010
int m;
int skt[N],tt;

int main()
{
    cin>>m;
    int x;
    for (int i = 0; i < m; i ++ ){
        scanf("%d", &x);
        //加入新输入的元素小于栈顶元素，那么该栈顶元素也将不会在未来的答案中出现
        //可以放心地把它删掉
        while(tt&&x<=skt[tt]) tt--;
        //如果到了栈底，说明前面所有数据都大于x
        if(!tt) {
            cout<<-1<<" ";
        }
        else cout<<skt[tt]<<" ";
        skt[++tt]=x;
    }
}

虽然有两层循环，但实际上由于每个元素最多只会入栈一次出栈一次，所以仍然是 $\Theta(n)$ .







2.3 KMP

    例题：给定一个模式串 $S$，以及一个模板串 $P$，所有字符串中只包含大小写英文字母以及阿拉伯数字。

模板串 $P$ 在模式串 $S$ 中多次作为子串出现。

求出模板串 $P$ 在模式串 $S$ 中所有出现的位置的起始下标。

输入格式:   第一行输入整数 $N$，表示字符串 $P$ 的长度。

  第二行输入字符串 $P$。

  第三行输入整数 $M$，表示字符串 $S$ 的长度。

第  四行输入字符串 $S$。

输出格式:

  共一行，输出所有出现位置的起始下标（下标从 0 开始计数），整数之间用空格隔开。

数据范围    $1\leq N\leq 10^5$

   $1\leq M\leq 10^6$

#include <iostream>
#define N 100010
#define M 1000010
using namespace std;

//s是模式串，P是模板串
char s[M],p[N];

//ne是next[]数组，用来回溯字符串
//ne[j]表示p的第j元素所对应的最长前后缀长度
int ne[N];

int main()
{
    int n,m;
    //+1是因为从下标1开始
    cin>>n>>p+1>>m>>s+1;
    
    //这一个循环体构建ne数组，本质上和匹配字符串一样，即自己和自己匹配。
    //从2开始，因为ne[1]肯定是0
    for (int i = 2,j=0; i <= n; i ++ ){
        //如果j不为0并且对应的字符不同，就回溯到上一位置
        while(j&&p[i]!=p[j+1]) j=ne[j];
        //相等就递增
        if(p[i]==p[j+1]) j++;
        //把递增的结果给ne
        ne[i]=j;
    }
    
    //匹配字符串，同理
    for (int i = 1,j=0; i <= m; i ++ ){
        while(j&&s[i]!=p[j+1]) j=ne[j];
        if(s[i]==p[j+1]) j++;
        if(j==n){
            //i-n是由于题目要求输出每个匹配的字符串的开始坐标。
            printf("%d ",i-n);
            //也要回溯，因为S中可能有别的匹配字符串
            j=ne[j];
        }
    }
    return 0;
}

    在暴力解法中，如果字符串 $p$ 某个位置与$s$ 不匹配，那么就把$p$从头和 $s$ 的下一位对比。这样时间复杂度会很高，$\Theta(n^2)$。

    KMP的基本思想就是类似动态规划的思想为了不做重复的计算（比较），事先会把每个字符位置上对应的最优解（最少的移动量）记录。

比如下图中上面是字符串$s$，下面是$p$，它们在第六个元素上不一样。暴力做法会让p的第一个A和s的第2个元素重新比较。KMP算法指出一点：假如字符串p中有公共前后缀，那么可以直接把前缀移动到后缀的位置，就可以节省很多步骤。公共前后缀的定义：如上图中两个白框所示，他俩就是一对公共前后缀：前缀以第一个元素开始，后缀以最后一个元素结尾的一模一样的两个子串。在移动之后就会变成这样：

注：前后缀的长度不可以等于p的长度，因为这样是没有意义的（即移动0次）。前后缀可以重叠，比如BABABC中，最长的前缀是p[1~3]，后缀是p[3~5]。这样就可以保证公共前后缀最长。而保证其最长（移动距离最短）的目的是不要跳过任何一个公共子串：

KMP的时间复杂度是$\mathcal{O}(2m)$，因为i共增加m次，导致j最多增加m次，而这一事实导致j最多减少m次。







2.4 Trie树

    Trie树是一种高效的存储/查找字符串集合的数据结构。它不是二叉树。

比如上面的图中，通过二叉树储存了6个字符串。每当插入一个字符串时，从根部的子节点开始插入第一个字符。如果字符存在，就顺着存在的节点往下走并继续插入。当某个字符不存在，则创建新子节点。有可能会出现一个字符串是另一个已经存放的字符串的前缀子串，那么会在某个子节点的位置（非叶节点）终止。

#include <iostream>
#define N 100010
using namespace std;

//idx:第x个输入；son：子节点数组;cnt:某个p位置上是否有字符串结束
int cnt[N],son[N][26];
int n,idx;
char str[N];
void insert(char str[]){
    //p：某个节点的idx
    //从根开始，所以初始化为0
    int p=0;
    for (int i = 0; str[i]; i ++ ){
        int u=str[i]-'a';
        if(!son[p][u]) son[p][u]=++idx;
        p=son[p][u];
    }
    cnt[p]++;
}

int query(char str[]){
    int p=0;
    for (int i = 0; str[i]; i ++ ){
        int u=str[i]-'a';
        if(!son[p][u]) return 0;
        p=son[p][u];
    }
    return cnt[p];
}

int main()
{
    cin>>n;
    while (n -- ){
        //使用字符数组，更保险,因为它可以忽略输入中的空格/回撤之类的无意义字符
        char op[2];
        scanf("%s%s", op, str);
        if(*op=='I'){
            insert(str);
        }else{
            printf("%d\n",query(str)); 
        }
    }
    return 0;
}






2.5 并查集

    并查集的两个作用：

  1. 把两个集合合并
  2. 询问两个元素是否在一个集合中

     每一个集合用一颗树来表示，树根的编号就是整个集合的编号。每个节点x存储它的父节点p[x]。

如何判断树根：如果是树根：p[x]==x

如何得到x的集合编号: while(p[x]!=x)x=p[x]; 这里需要$\mathbb{O}(n)$，通过路径压缩为几乎 $\mathbb{O}(1)$

如何合并集合：直接p[x]=y，把根节点嫁接到另一个集合的结点上，作为子节点存在。



2.5.1 合并集合

  一共有 n 个数，编号是 1∼n，最开始每个数各自在一个集合中。

  现在要进行 m 个操作，操作共有两种：

  M a b，将编号为 a 和 b 的两个数所在的集合合并，如果两个数已经在同一个集合中，则忽略这个操作；

  Q a b，询问编号为 a 和 b 的两个数是否在同一个集合中；

输入格式：   第一行输入整数 n 和 m。

  接下来 m 行，每行包含一个操作指令，指令为 M a b 或 Q a b 中的一种。

输出格式：   对于每个询问指令 Q a b，都要输出一个结果，如果 a 和 b 在同一集合内，则输出 Yes，否则输出 No。

每个结果占一行。

数据范围:

$1\leq n,m\leq 10^5$ 输入样例：

4 5
M 1 2
M 3 4
Q 1 2
Q 1 3
Q 3 4

输出样例：

Yes
No
Yes

  代码：

//储存父节点idx
int p[N];
//这个函数查找a的祖宗节点
int find(int a){
    //让a的父节点直接等于其根节点
    //也就是路径压缩
    if(p[a]!=a) p[a]=find(p[a]);
    return p[a];
}

int main()
{
    int n,m;
    scanf("%d%d", &n, &m);
    //初始化，让每个节点的父节点等于自己
    for (int i = 1; i <= n; i ++ ) p[i]=i;
    char op[2];
    int a,b;
    while (m -- ){
        scanf("%s%d%d", op,&a,&b);
        if(*op=='M') p[find(a)]=find(b);
        else{
            if(find(p[a])==find(p[b])) printf("Yes\n");
            else printf("No\n");
        }
    }
    return 0;
}

注：这个例题中的“集合”是通过M慢慢增长的。最开始的时候数组p的每个元素的父节点都是它自己，但是在第一次合并集合之后，有两个元素变成了一个集合，其中一个是另一个的父节点/祖宗节点。路径压缩让find几乎为常数复杂度是因为每次第一次为一个元素调用find, 需要要回溯。



2.5.2 连通块中点的数量

给定一个包含 n 个点（编号为 1∼n）的无向图，初始时图中没有边。

现在要进行 m 个操作，操作共有三种：

C a b，在点 a 和点 b 之间连一条边，a 和 b 可能相等；

Q1 a b，询问点 a 和点 b 是否在同一个连通块中，a 和 b 可能相等；

Q2 a，询问点 a 所在连通块中点的数量； 输入格式 第一行输入整数 n 和 m。

接下来 m 行，每行包含一个操作指令，指令为 C a b，Q1 a b 或 Q2 a 中的一种。

输出格式 对于每个询问指令 Q1 a b，如果 a 和 b 在同一个连通块中，则输出 Yes，否则输出 No。

对于每个询问指令 Q2 a，输出一个整数表示点 a 所在连通块中点的数量

每个结果占一行。

数据范围 $1 \leq n,m\leq 10^5$

#include <iostream>
#define N 100010
using namespace std;

//cnt：当前元素的祖宗节点有多少个元素
int cnt[N], p[N];

int find(int a){
    if(a!=p[a]) p[a]=find(p[a]);
    return p[a];
}

int main()
{
    int n,m;
    cin>>n>>m;
    for (int i = 1; i <= n; i ++ ) {
        //初始化父元素和计数器
        p[i]=i;
        cnt[i]=1;
    }
    string op;
    int a,b;
    while (m -- ){
        cin>>op;
        if(op=="C"){
            scanf("%d%d", &a, &b);
            a=find(a);
            b=find(b);
            //即两者不在同一个集合才需要考虑集合大小增加
            //没有这个判断会造成数量重复增加 
            if(a!=b){
                p[a]=b;
                cnt[b]+=cnt[a];
            }
        }
        if(op=="Q1"){
            scanf("%d%d", &a, &b);
            if(find(a)==find(b)) printf("Yes\n");
            else printf("No\n");
        }
        if(op=="Q2"){
            scanf("%d\n", &a);
            printf("%d\n",cnt[find(a)]);
            
        }
    }
    return 0;
}






2.5 堆

    堆是完全二叉树。这一节通过一位数组模拟一个堆。这里是用的是小根队，即每个节点小于等于子节点，所以跟节点就是最小值。根节点在h[1]，左节点h[2]，右节点h[3], p[x]节点左节点p[2x]，右节点p[2x+1]。

    重要的函数：down(n):观察n节点和其2个子节点。假如n节点的值大于左右节点，那么左右节点最小的和n切换。up(n):看n节点的父节点，如果小于其父节点，那么就交换。

    时间复杂度：添加元素：$\Theta(\log n)$，删除元素$\Theta(\log n)$，修改元素$\Theta(\log n)$，最小值$\Theta(1)$。

    构建堆：可以为每一个调用插入实现对于堆的构建，那样时间复杂度为 $\Theta(n\log n)$；但是存在一种优化：从n/2开始down，$\Theta(n)$



2.5.1 堆排序

    输入n个数字，使用堆排序。

using namespace std;

const int N = 100010;

int n, m;
int h[N], cnt;

//由于不需要插入特定位置操作，不需要使用up函数
//传入的参数是一个index
void down(int u)
{
    //t是u和其两个子节点之中最小的节点
    int t = u;
    if (u * 2 <= cnt && h[u * 2] < h[t]) t = u * 2;
    if (u * 2 + 1 <= cnt && h[u * 2 + 1] < h[t]) t = u * 2 + 1;
    if (u != t)
    {
        swap(h[u], h[t]);
        //递归调用down，因为上一行的swap可能仍然会导致小根堆规则被破坏
        down(t);
    }
}

int main()
{
    scanf("%d%d", &n, &m);
    //先把输入插入堆中
    //把1看作根节点，因为0*2还是0
    for (int i = 1; i <= n; i ++ ) scanf("%d", &h[i]);
    cnt = n;

    //从n/2开始调用down，一直调用到根节点，实现小根堆建构
    for (int i = n / 2; i; i -- ) down(i);

    while (m -- )
    {
        printf("%d ", h[1]);
        //直接让最后一个元素放到对顶
        //cnt--把原最后一个元素排除在外，因为它已经在对顶了
        h[1] = h[cnt -- ];
        //重新排序，保证堆顶是最小值
        down(1);
    }

    puts("");

    return 0;
}


2.5.2 模拟堆

    需求：

I x，插入一个数 x； PM，输出当前集合中的最小值； DM，删除当前集合中的最小值（数据保证此时的最小值唯一）； D k，删除第 k 个插入的数； C k x，修改第 k 个插入的数，将其变为 x；

#include <iostream>
#include <algorithm>
#include <string.h>

using namespace std;

const int N = 100010;
//ph[k]:第k个输入的元素对应的index: pointer to heap's index
//hp[l]:index=l的元素是第几个输入的: heap's index to pointer
//h:heap
//cnt：当前存在的元素总个数（堆底index），用来维护新元素的下标
int h[N], ph[N], hp[N], cnt;

//由于有对“第k个插入的元素”进行修改的要求，需要知道它是谁
//由于想要知道它是谁，就要维护ph与hp
void heap_swap(int a, int b)
{
    //必须要知道index为a的元素是第几个输入的
    //才能通过k找到第k个输入的元素对应下标。
    swap(ph[hp[a]],ph[hp[b]]);
    swap(hp[a], hp[b]);
    swap(h[a], h[b]);
}

//传入的参数是一个index
void down(int u)
{
    //t是u和其两个子节点之中最小的节点
    int t = u;
    if (u * 2 <= cnt && h[u * 2] < h[t]) t = u * 2;
    if (u * 2 + 1 <= cnt && h[u * 2 + 1] < h[t]) t = u * 2 + 1;
    if (u != t)
    {
        heap_swap(u, t);
        down(t);
    }
}

void up(int u)
{
    //循环调用，直到u==1，即根节点
    while (u / 2 && h[u] < h[u / 2])
    {
        heap_swap(u, u / 2);
        u >>= 1;
    }
}

int main()
{
    //m:用来维护“第k个插入的元素”
    int n, m = 0;
    scanf("%d", &n);
    //不同于2.5.1，这里的堆构建是通过I指令插入，所以不能整个排序
    while (n -- )
    {
        char op[5];
        //k：第k个插入的元素；x：值
        int k, x;
        scanf("%s", op);
        //插入
        if (!strcmp(op, "I"))
        {
            scanf("%d", &x);
            //先把这个新元素放在末尾
            cnt ++ ;
            m ++ ;
            ph[m] = cnt, hp[cnt] = m;
            h[cnt] = x;
            //使用up排序，cnt永远是堆底
            up(cnt);
        }
        else if (!strcmp(op, "PM")) printf("%d\n", h[1]);
        else if (!strcmp(op, "DM"))
        {
            //把堆顶和堆底调换，并删除堆底（原堆顶）然后排序
            heap_swap(1, cnt);
            cnt -- ;
            down(1);
        }
        else if (!strcmp(op, "D"))
        {
            scanf("%d", &k);
            //得到第k个输入的元素的下标
            k = ph[k];
            //交换堆底和k
            heap_swap(k, cnt);
            //删除堆底
            cnt -- ;
            //对原堆底元素进行排序
            //up和down只会运行一个（函数体内的判断决定）
            up(k);
            down(k);
        }
        //修改第k个元素
        else
        {
            scanf("%d%d", &k, &x);
            //不存在删除所以不需要swap
            //直接修改即可
            k = ph[k];
            h[k] = x;
            up(k);
            down(k);
        }
    }

    return 0;
}


2.6 哈希表

    哈希函数把一个巨大的数据范围映射到一个较小的数据范围。离散化就是一种特殊的哈希方式，但本节的哈希指的是一般的哈希方式。哈希表通常有两种储存结构：开放寻址法和拉链法。它们用不同的方式解决同一个哈希函数可能会把两个不同的数字映射成一个数字的情况。选择一种使用即可。

    简单的哈希函数通常只是把输入的数据取mod，这个模的值需要是一个质数并且越远离2的整次幂越好。哈希表的所有操作基本上都是 $\mathcal{O}(1)$。



2.6.1 哈希方法

    拉链法。给哈希数据范围创建一个数组，每个位置都代表一个哈希过后的值。假设11和23都在某个哈希函数之后映射到了3，那么在3的位置“拉出一个链表”，储存11和23。这里的哈希数组相当于一个链表的数组，每个数组的元素都是一个链表头。哈希算法是一种期望算法，每个链表的长度平均来看都很短，其操作可以看做常数。通常哈希表只有添加和查找的操作：使用哈希函数找到对应的数组位置然后遍历即可。删除操作基本不做要求，如果有删除操作的话，直接通过给元素设置一个布尔属性，标记为是否被删除，而非真正的删除。

//100003是大于十万的最小的质数
#define N 100003

//h:代表映射后的数组 k:代表映射后的值
//e,ne是数组
int h[N],e[N],ne[N],k,idx;

void insert(int v){
    //在C++中，负数取模是负数，通过这样的变换变成真正的整数模
    k= (v%N+N)%N;
    //头插法：把新数据插在指定的映射后数组的头部
    e[idx]=v;
    ne[idx]=h[k];
    h[k]=idx++;
}

bool find(int v){
    k=(v%N+N)%N;
    //从映射后数组的头部开始遍历
    for (int i = h[k]; i !=-1; i=ne[i] ){
        if(e[i]==v) return true;
    }
    return false;
}

int main()
{
    int n=0;
    //把h数组初始化为-1，因为所有尾节点都是-1
    memset(h, -1, sizeof h);
    cin>>n;
    char op[2];
    int v;
    while (n -- ){
        scanf("%s%d", op, &v);
        if(*op=='I') insert(v);
        else{
            if(find(v)) puts("Yes");
            else puts("No");
        }
    }
    return 0;
    
}

    开放寻址法。开放寻址法类似停车场中找停车位：假设有一个只有一排的的停车场，通过函数决定新进来的车的目标编号，然后把车停到目标的停车位。由于还想函数会生成一样的编号，每当冲突的时候这个车就需要从其目标编号开始向后寻找可用的停车位。为了减少冲突，我们开数据量的两到三倍的值（经验值）作为停车场中停车位的数量（同样最好是质数）。 同样地，开放寻址法也不去关心真正的删除，只会把目标元素改成一个非法的值或者用其他方法标记。

using namespace std;
//取数据范围的2-3倍
#define N 200003
//0x3f3f3f3f是最大值（大于10^9，考虑为最大值）
#define null 0x3f3f3f3f
//开放寻址法只需要一个数组
int h[N],k;

//这个方法供插入和查询使用
int find(int v){
    k=(v%N+N)%N;
    //假如h[k]==null,说明找到了可用的槽位/没找到目标数据
    //假如h[k]==v，说明找到了目标值
    while(h[k]!=null&&h[k]!=v){
        k++;
        //如果到头了，就从0开始
        if(k==N) k=0;
    }
    return k;
}

int main()
{
    int n;
    cin>>n;
    //把映射后的数组设置成null
    //因为memset每次设置1Byte，所以这里是3f
    memset(h,0x3f,sizeof(h));
    while (n -- ){
        char op[2];
        int v;
        scanf("%s%d",op,&v);
        if(*op=='I') {
            h[find(v)]=v;
        }else {
            if(h[find(v)]==null) printf("No\n");
            else printf("Yes\n");
        }
    }
    return 0;
}


2.6.2 字符串哈希

    假设一道题目要求判断一个字符串中任意选择的子串是否完全一样。使用暴力做法可以得到$\mathcal{O}(n)$ 的结果，可能超时。而字符串哈希可以达到$\mathcal{O}(1)$。其主要的理念是把一个字符串的每一个前缀映射为一个P进制的数字并且使用一个较小的数字Q来取模得到其哈希值。比如存在一个字符串 $ABCDEFG$，那么它可以被映射为(这里假设A-Z从1递增，而实际上会采用ASCII)：

映射数组[i]= 子串: 子串对应的哈希值 h[0]= 0; 通常不会让任何一个字符的值为0，因为无法区分出多个该字符的重复：000和00都是0. h[1]= A: 1 h[2]= AB: 12 h[3]= ABC: 123 h[4]= ABCD: 1234 ……

通常来说，这个$P$ 是 $131$ 或者 $13331$ （上面的例子中使用10进制，但实际上10进制不能用）。这是一个经验值，认为这两个值可以达到最小的冲突概率。而$Q$通常是$2^{64}$，这个值可以直接通过转换为unsigned long long 来直接取成模。

比如上图我们希望求出 $L~R$ 之间的字符串子串的哈希值，我们需要先知道L与R的哈希值。在之前的例子中，我们可以通过 ABCD和AB计算出CD的哈希值。在这个场景中，ABCD=1234，AB=12，所以CD=ABCD-AB00=1234-12*10^2=34。可以把这个思路推广成一个公式: ans=h[R]-h[L-1]*p[R-L+1]，其中R与L分别是目标子串的左右端点，p数组是预处理出的“P进制下，把L左移R-L+1所需的数字”。注：这个例子数据小，不存在取模造成的数据丢失，所以可以判断出CD=34，但是一旦有模存在，这种对应关系不存在。

//使用131作为P
#define N 100010
#define P 131
typedef unsigned long long ULL;
//当传入数组的时候，自动取模，p数组也要取
ULL p[N],h[N];
char str[N];

ULL get(int l, int r){
    //由于任意映射后的长度最多为63位，所以p中的数字也是0～63位
    //这就是p中的数字也需要取模的原因：每当h中的数字大于63位便会变成小于等于63位的数
    //      此时意味着对应着的p也会被限制在这一范围
    return h[r]-h[l-1]*p[r-l+1];
}


int main()
{
    int n,m;
    cin>>n>>m;
    scanf("%s",str+1);
    p[0]=1;
    //h赋值同时哈希，p预处理
    for (int i = 1; i <= n; i ++ ){
        h[i]=h[i-1]*P+str[i];
        p[i]=p[i-1]*P;
    }
    while (m -- ){
        int l1,r1,l2,r2;
        scanf("%d%d%d%d", &l1, &r1,&l2,&r2);
        if(get(l1,r1)==get(l2,r2)) puts("Yes");
        else puts("No");
    }
    return 0;
    
}

两个不同的子串实际上也可能是会相等的，但是概率太小，不考虑。这里有些细节值得思考：整个算法中我们考虑的只是映射后的哈希值。这种映射关系是非对称的，即无法从哈希值反推实际字符串，而我们实际上也并不关心映射后的字符串。每当某两个h[r]-h[l-1]*p[r-l+1]之差相等，我们就可以说它们是一样的子串：而这两个子串是什么，我们并不关心。同时也要注意，“左移”这一行为被数组p预处理，但实际上由于p本身也存在取模，“左移”在第一次达到取模的情况后便不是真正意义上的左移了。与其理解为左移，不如理解为“当2个区间长度确定时，为了保证区间长度一样，p作为一个编码存在。h[l-1]*这个编码 降低了哈希重复的可能。”（长度不一样结果不一样）
