导言：

    本章介绍动态规划中的背包问题，区间DP，线性DP，计数DP，数位统计DP，状态压缩DP，树形DP，和记忆化搜索。     动态规划的思考方式：通过集合的思想

状态表示：思考整个问题需要用几维来表述，每一个状态的含义是什么，每一种状态都是一个集合；集合：状态表示的集合是什么；属性：集合的目标属性是什么（Min,Max,Num），在矩阵里实际上就是值；状态计算：如何一步一步把每个元素算出来，对应的是集合的划分：集合的划分的目的是“考虑如何把当前的集合，划分成若干更小的子集，使得这些子集我们都可以算出来/可以从前面的更小的状态中直接使用”：

以背包问题为例，集合 $f(i,j)$ （矩阵坐标）可以被看作 “物品个数$\leq i$，背包容积$\geq j$ ” 的情况下的集合，而它的值（属性）为最大值，也就是 “物品个数$= i$，背包容积$= j$ ”时背包中价值最大值。通过Fig.2 可得 最大值 $= max(f(i-1,j), f(i-1,j-v[i])+w[i])$，其中 $v,w$ 分别为代表物品体积和价值的数组。上图中把一个“当前集合”划分成了两个子集，即不考虑当前物品i，以及考虑当前物品i。先然后两个子集的并集是当前集合。右侧子集实际上也使用了更小的子集：即不考虑物品i再加上物品的i价值=包含物品i的集合

    DP问题的优化通常是对代码或者方程做一个等价变形，考虑的时候从最简单的来，然后再优化；在遍历中如果使用i-1这样的角标，那么i最好从1开始，并且给i=1设置边界值；DP的复杂度计算通常是 状态*状态转移次数







5.1 背包问题

    背包问题中，给定一个容积为 $V$ 的背包与 $n$ 件价值 $w$ 不同，体积 $v$不同的物品。 背包问题分成几大类：

  1. 0/1背包：每个物品只能最多装入一次
  2. 完全背包：假设每个物品有无限个复制，可以无限添加1个物品
  3. 多重背包：同一个物品个数有限，在个数上限之中可随意添加
  4. 分组背包：所有物品被分配到 $a$ 个分组中，只能添加每个分组中的某一个物品


5.1.1 0/1背包

    0/1背包的核心在于比较“在背包容积确定的情况下，不考虑当前物品 $i$ 的价值和考虑当前物品的价值所得的价值”哪个更高。前者实际上是当物品数量为 $i-1$ 时的最高价值。后者实际上是“当前物品价值”加上“在考虑当前物品的体积之前的背包容积中所能承载的 $i-1$ 个物品的价值上线” 的值。

#define N 1010
int v[N],w[N];
int f[N][N];
int m,n;

int main(){
    //输入 n：物品数量，m：背包容积
    cin>>n>>m;
    for (int i = 1; i <=n; ++i) {
        cin>>v[i]>>w[i];
    }
    //当i=0，没有价值，所以直接跳过
    for (int i = 1; i <= n; ++i) {
        for (int j = m; j >=v[i] ; --j) {
            //当这个物品的体积超过了当前容积时，不执行
            //如果没有这个判断，可能导致访问到垃圾内容（负数坐标）
            if(j>=v[i])
            //对应着上面提到的两点
            f[i][j]= max(f[i-1][j],f[i-1][j-v[i]]+w[i]);
        }
    }
    cout<<f[n][m]<<endl;//最右下角的数据意为“当有n个物品，容积上限为m”时的总价值
}

     上面这个算法使用了二维数组，但实际上0/1背包问题可以被优化为一维数组：

int f[N];
int main(){
cin>>n>>m;
    for (int i = 1; i <=n; ++i) {
        cin>>v[i]>>w[i];
    }
    for (int i = 1; i <= n; ++i) {
        for (int j = m; j >=v[i] ; --j) {
            f[j]= max(f[j],f[j-v[i]]+w[i]);
        }
    }
    cout<<f[m]<<" ";
}

最重要的改变是循环体的内容f[j]= max(f[j],f[j-v[i]]+w[i]);。观察二维算法，从始至终只有f[i-1][a],f[i][a]被使用，说明这里可以使用滚动数组。也就是说，由于我们不需要去记忆f[i-1]之前的任何内容，f[j]= max(f[j],f[j-v[i]]+w[i]);中的max里的f[j]实际上记录的是f[i-1][j]，而假设后一项大于该项，f[j]就会被更新为当前的最大值，原来的f[i-1][j]的值就被遗忘了。

    再观察f[j-v[i]]+w[i]可以看到f[j-v[i]]本身指的是“找到不考虑当前物品的体积时的体积所对应的最大价值”，+w[i]得到考虑当前物品价值的最大价值：这和二维算法差不多。但一个不同是，“找到不考虑当前物品的体积时的体积所对应的最大价值”中的第二个“体积”可能已经考虑了当前物品了：这种情况下对应的实际上是f[i][j-v[i]]而不是f[i-1][j-v[i]]。这也是因为j从小到大排序，所以j-v[i]是在j之前被计算的一个状态（但都是在i=同一个值的时候，可以叫做“同一层”）。这种情况下可能会导致这个物品再次被考虑（比如j>2v[i]）。

     解决方法就是，倒序遍历j。这样一来，相当于每次都考虑首先“当物品有i个，背包上限为m时”的价值：在更新f[j]前这个变量里保存的实际上是上一次外部循环（i-1）所得到的值，而这个值必然大于等于其他任何f[j-v[i]], 并且还没有考虑“当前物品”（也就是说在刚进入新一轮的对i的循环时，f数组中的值都还没有考虑i，这使得倒序遍历是安全的）。

     这里体现的就是对状态转移方程的等价变形。

    思考这样一个例子输入：

//n=5,m=20
5 20
1 2
2 4
3 6
4 8
5 10

假如当前有5个物品（上面的输入是 “体积 价值”），并且背包内容的上限是20。如果使用正序遍历，那么会得到这样的结果：（输出方法为：在每个f[j]= max(f[j],f[j-v[i]]+w[i]);后面输出更新后的f[j]，并且在下一次外部循环开始前换行）

//x仅代表占位对齐

以第一行为例，它是“只有一个物品，该物品体积为1，价值为2，背包容量j从1增长到m”的结果。由于它的体积为1，每一次m的增加都相当于增加了它的倍数。在正序遍历中会导致f[j-v[i]]+w[i]中的f[j-v[i]]得到一个考虑过当前物品的情况（除了最开始j=1），而+w[i]又重复添加了一次。后面几行也是一样的，每到当前i的倍数的时候，就会重复考虑。

    但是如果是倒序，输出则是这样的：

很显然不会造成上述的重复，并且即使空间超过每个物品之和也不会造成重复考虑，因为f[j-v[i]]+w[i];中f[j-v[i]]得到的永远是还没有考虑当前物品加入情况的值。



5.1.2 完全背包问题

    在完全背包中，一个物品可以无限次地被添加，这就导致了对于子集的比较变成了（其实和0/1是类似的）：1.不考虑当前物品：f[i]][j]=f[i-1][j] 2.添加1个当前物品：f[i]][j]=f[i-1][j-v[i]]+w[i] 3. 添加两个当前物品：f[i]][j]=f[i-1][j-2*v[i]]+2*w[i] 4. 添加三个当前物品：f[i]][j]=f[i-1][j-3*v[i]]+3*w[i]…… k+1. 添加k个当前物品：f[i]][j]=f[i-1][j-k*v[i]]+k*w[i]。仍旧，目标是从这么多枚举中找出价值最大的一个，可以通过循环进行比较。

#define N 1010
int m,n;
int f[N][N];
int v[N],w[N];

int main(){
    cin>>n>>m;
    for(int i=1;i<=n;i++) cin>>v[i]>>w[i];

    for(int i=1;i<=n;i++)
        for(int j=0;j<=m;j++)
            for(int k=0;k*v[i]<=j;k++){
                //这里考虑0个当前物品==f[i]][j]=f[i-1][j]，不需要专门写了
                f[i][j]=max(f[i][j],f[i-1][j-k*v[i]]+k*w[i]);
            }    
    cout<<f[n][m];
    return 0;
}

在朴素代码实现中，使用了三重循环，这导致时间复杂度过高：$O(n*m^2)$，所以需要优化。尝试展开f[i][j]=max(f[i][j], f[i-1][j-k*v[i]]+k*w[i])，可以得到：

        f[i][j]=max(f[i][j], f[i-1][j-1*v[i]]+1*w[i], f[i-1][j-2*v[i]]+2*w[i]),...

对f[i][j-v[i]]做相同的展开可以得到:

        f[i][j-v[i]]=max(f[i][j-v[i]], f[i-1][j-2*v[i]]+1*w[i], f[i-1][j-3*v[i]]+2*w[i]),...

观察上下两个展开，可以看出，在对应的每一项中，f[i][j]以w[i]大于f[i][j-v[i]]。这就意味着，前者的最大值比后者的最大值也大w[i]。这样就可以把k循环优化掉：

#define  N 1010
int n,m;
int v[N],w[N];
int f[N][N];

int main(){
    cin>>n>>m;
    for(int i=1;i<=n;i++) cin >> v[i]>>w[i];

    for(int i=1;i<=n;i++)
        //这里必须从0开始，因为要初始化f[i][0~v[i]-1]之间的所有值供f[i][v[i]]使用
        for(int j=0;j<=m;j++){
            //注：上面的朴素算法没有考虑这行是因为k=0等价于这行
            f[i][j]=f[i-1][j];
            //上一行必须存在，不然这左项就可能是0了
            //注：这里是f[i][j-v[i]]+w[i]而不是f[i-1][j-v[i]]+w[i]
            //这里必须要有if(j>=v[i]) ，不然超出数组边界
            if(j>=v[i]) f[i][j]=max(f[i][j],f[i][j-v[i]]+w[i]);
        }
    cout<<f[n][m];
    return 0;
}

     最后，完全背包问题也一样可以被优化为一维数组：

#define  N 1010
int n,m;
int v[N],w[N];
int f[N];
int main(){
    cin>>n>>m;
    for(int i=1;i<=n;i++) cin >> v[i]>>w[i];

    for(int i=1;i<=n;i++)
        for(int j=v[i];j<=m;j++){
            //必须从v[i]开始，因为假如编译器把数组超出边界后
            //的值看作0，那么会出现f[j]和w[i]对比的情况出现
            //当j<v[i],这种对比可能造成“背包空间不够i的体积，
            //但还是把i的价值进行了赋值”
            f[j]=max(f[j],f[j-v[i]]+w[i]);
        }
    cout<<f[m];
    return 0;
}

完全背包的一维优化和0/1背包基本一致，唯一不同的就是，我们不需要去关心“假如从小到大遍历，会不会导致重复计算”，因为完全背包允许重复 。

     并且，可以注意到f[j-v[i]]+w[i]所得到的值并不一定是完全在i层，因为f[j-v[i]]在j<2v[i]的情况下，访问的实际上是i-1的数据。这并不影响，因为在j<2v[i]时，1. 从代码逻辑的角度上说：未优化的版本也使用的也是上层的数据（f[i][j]=f[i-1][j]），2.从理解的逻辑上来说：实际上表达的等同于“不考虑当前物品的价值+考虑当前物品的价值”。

注：max中的f[j]在第一次判断的时候实际上就是f[i-1][j]了，所以不用担心会缺少对于“不选该物品”的考量。



5.1.3 多重背包问题

    多重背包问题和完全背包问题其实是十分相似的。相较于后者假设一个物品可以装入背包无限次，前者规定了每个物品的数量。所以在考虑当前背包容量收否能装下物品的同时，也要考虑物品有没有剩余了。 所以可以在朴素完全背包的基础上获得多重背包的朴素算法

#define N 110

int n,m;
//二维数组未优化
int f[N][N];
//3数组分别接收 体积，价值，个数
int v[N],w[N],s[N];

int main()
{
    cin>>n>>m;
    for (int i = 1; i <= n; i ++ )
        cin>>v[i]>>w[i]>>s[i];
        
    for (int i = 1; i <= n; i ++ )
        for (int j = 0; j <= m; j ++ )
            //这里和完全背包问题完全不一样的地方只是对k的判断！
            for (int k = 0; k <= s[i]&&k*v[i]<=j; k ++ )
                f[i][j]=max(f[i][j],f[i-1][j-k*v[i]]+w[i]*k);
    cout<<f[n][m];
    return 0;
       
}

    由于存在三重for嵌套，时间复杂度约为$NVS$. 之后可以尝试着去优化这个算法。可以和之前一样观察两个展开式：

展开f[i][j]=max(f[i][j], f[i-1][j-k*v[i]]+k*w[i])得：

        f[i][j]=max(f[i][j], f[i-1][j-1*v[i]]+1*w[i], f[i-1][j-2*v[i]]+2*w[i]),...,f[i-1][j-s*v[i]]+s*w[i])

对f[i][j-v[i]]做相同的展开可以得到:

        f[i][j-v[i]]=max(f[i][j-v[i]], f[i-1][j-2*v[i]]+1*w[i], f[i-1][j-3*v[i]]+2*w[i]),...,f[i-1][j-s*v[i]]+(s-1)*w[i]), f[i-1][j-(s+1)*v[i]]+s*w[i])

     观察两者省略号后面的这部分：f[i-1][j-s*v[i]]+s*w[i])和f[i-1][j-s*v[i]]+(s-1)*w[i]), f[i-1][j-(s+1)*v[i]]+s*w[i])。可以看到前者很正常，表示为“当这个物品的所有个数都被考虑的价值”，类似完全背包中的最后一项“当这个物品尽可能多地放入后的价值”。而后者有些反直觉，比如出现了(s+1)。造成这一项的原因是因为当前所考虑的物品的数量仍然是s，但是它在最开始就已经-v[i]了，这导致倒数第二项f[i-1][j-s*v[i]]+(s-1)*w[i])其实还少考虑1个该物品来让它达到s个，故加一。

    也就是因为后者最后一项的出现，使得它无法像完全背包一样考虑最大值。首先要知道，因为前者的式子中是不存在后者最后一项中对应的式子的，这就说明在考虑最大值的时候，后者的最后一项应当不被考虑。可这导致，尽管我们可以知道后者的最大值，我们无法知道排除掉最后一项之后的最大值。举个例子，设有集合中存在10个数字，一个函数可以计算这个集合中的最大值，问：当有人从其中随机取出一个数字之后，可以通过函数之前给出的结果直接得到取出后的最大值吗？答案显然是否定的：最乐观的情况就是取出的值不是最大值，导致最大值不变，但是在这个例子中并无法保证该假设。所以该方法弃用。当然，你可以说重新计算取出后的集合的最大值，但那和动态规划不去做重复计算的思想相悖。

    多重背包问题使用的优化方式叫做 二进制优化。二进制优化通过二进制的思想把某个有n个的物品分组成 $\lceil\log_2n \rceil$ 组。假设一个物品有200个体积为v价值为w，那么它可以表示为1+2+4+8+16+32+64再加上73的8个组，也就是 $\lceil\log_2200 \rceil$。73不是200的二进制表达中的医院，这是因为前几项的和是127，这使得前7项可以表达从0~127的任意数量。而假如使用128，二进制表达的下一项而非73，那么最多可以表达255，超出了上限200.但是使用73的话，就可以正好取到200了。现在实际上这个物品被分成了8个物品，体积分别是1*v, 2*v ..., 64*v, 73*v，价值分别是1*w, 2*w ..., 64*w, 73*w：对每一个物品做这样的优化，一个多重背包问题就被转换成了一个更长的0/1背包问题。由于进行了这种拆分，装数据的数组也会变长：假设数据范围 $$0<N≤1000, 0<V≤2000, 0<v_i,w_i,s_i≤2000$$

表示有N种物品，V背包容积，三个整数 $v_i,w_i,s_i$分别表示第 i 种物品的体积、价值和数量。那么再假设每个物品的个数都达到了上限，优化之前的遍历次数最多为40亿，而现在时间复杂度约为$NVlog_2S$，遍历次数最多2200万。


//这个N是log_2S （为了保险，凑到25000）
#define N 25000
int n,m;
int f[N];
int v[N],w[N];

int main()
{
    //读入物品种类数量和背包容积
    cin>>n>>m;
    //a：物品体积；b：物品价值；s：物品个数；
    //k：优化之后当前这一组的个数；cnt：计数，作为优化后的数组的角标
    int a,b,s,k,cnt;
    cnt=0;
    for (int i = 1; i <= n; i ++ ) {
        //每一个物品的第一组的初始化值都是1
        k=1;
        cin>>a>>b>>s;
        //检查k是否大于s，不让像上面128的情况出现
        while(k<=s){
            cnt++;
            //优化后这一组物品的价值和体积
            v[cnt]=k*a;
            w[cnt]=k*b;
            //减去当前组的个数
            s-=k;
            //倍增k
            k*=2;
        }
        //当s大于0，说明出现了需要向上面+73的行为
        if(s>0){
            cnt++;
            v[cnt]=s*a;
            w[cnt]=s*b;
        }
    }
    //让n等于更新后的“物品个数”
    n=cnt;
    
    //进行一个0/1背包即可
    for(int i=1;i<=n;i++)
        for (int j = m; j >=v[i]; j -- )
            f[j]=max(f[j],f[j-v[i]]+w[i]);
            
    cout<<f[m];
    return 0;
    
}


5.1.4 分组背包问题

    线性DP指的是在使用DP的时候，对于子问题的使用有一个线性关系的，就叫线性DP。比如背包问题使用二维矩阵，每一层DP实际上都是从左往右线性地一行一行地进行，它就是一种线性DP。



5.2.1 数字三角形

    给定一个如下图所示的数字三角形，从顶部出发，在每一结点可以选择移动至其左下方的结点或移动至其右下方的结点，一直走到底层，要求找出一条路径，使路径上的数字的和最大。

        7
      3   8
    8   1   0
  2   7   4   4
4   5   2   6   5

输入格式
第一行包含整数 n，表示数字三角形的层数。

接下来 n 行，每行包含若干整数，其中第 i 行表示数字三角形第 i 层包含的整数。

输出格式
输出一个整数，表示最大的路径数字和。

    首先还是使用集合来分析。这里的状态表示仍是二维f[i][j]，代表着达到这个坐标的所有从上到下的路径，属性为路径最大值。状态计算中，可以把这个集合划分为两个部分：从左上来，从右上来。

int f[N][N],g[N][N],n;

int main()
{
    //读入行数
    cin>>n;
    //初始化转移矩阵为-INF
    //从0开始为的是对于三角左边的左上的边界处理
    for (int i = 0; i <= n; i ++ )
        //i+1为的是对于三角右边的右上的边界处理
        for (int j = 0; j <= i+1; j ++ )
            f[i][j]=-INF;
    
    //输入三角
    for (int i = 1; i <= n; i ++ )
        for (int j = 1; j <= i; j ++ )
            scanf("%d", &g[i][j]);
            
    //顶点确定
    f[1][1]=g[1][1];
    
    //直接从第二行开始，遍历每一行
    for (int i = 2; i <= n; i ++ )
        //遍历每一行的每一个元素
        for (int j = 1; j <= i; j ++ )
            //对比左上和右上，且考虑当前值的最大值
            f[i][j]=max(f[i-1][j-1]+g[i][j],f[i-1][j]+g[i][j]);
            
    int m=-INF;
    //遍历最后一行
    for (int i = 1; i <= n; i ++ )
        m=max(m,f[n][i]);
        
    cout<<m;
}


5.2.2 最长上升子序列

    给定一个长度为 N 的数列，求数值严格单调递增的子序列的长度最长是多少。

    状态表示：f[i],表示“以第i个元素结尾的子序列”的集合，属性为最长长度；计算时的集合划分为“比第i个元素小1～i-1的元素”的最长长度。

int main()
{
    cin>>n;
    for (int i = 1; i <= n; i ++ ) scanf("%d", &a[i]);
    
    for (int i = 1; i <= n; i ++ ){
        //只包含其本身
        f[i]=1;
        //遍历之前的f
        for (int j = 1; j < n; j ++ ){
            //找到比a[i]小的元素，查看其f
            if(a[i]>a[j]) f[i]=max(f[i],f[j]+1);
         }
    }
    int res=0;
    //达到最大值
    for (int i = 1; i <= n; i ++ ) res=max(res,f[i]);
    
    cout<<res;
}

复杂度：$O(n^2)$。

注：动态规划的题目中如果想要复原路径，那么把状态转移记录下来就好了。

优化：$O(n\log n)$.

int main()
{
    cin>>n;
    for (int i = 0; i < n; i ++ ) scanf("%d", &a[i]);
    
    int len=0;
    
    for (int i = 0; i < n; i ++ ){
        int l=0, r=len;
        while(l<r){
            int mid=l+r + 1>>1;
            if(g[mid]<a[i]) l=mid;
            else r=mid-1;
        }
        len=max(len, r+1);
        g[r+1]=a[i];
    }
    for (int i = len-1; i >= 0; i -- ) cout<<g[i];
}


5.2.3 最长公共子序列

    给定两个长度分别为 N 和 M 的字符串 A 和 B，求既是 A 的子序列又是 B 的子序列的字符串长度最长是多少。

    这道题的状态表示为f[i][j]，代表着第一个字符串前i个字符，第二个字符串前j个字符可以组成的最长公共子串。集合划分：不选i但一定有j；不选j但一定有i；都不选；都选。后两者很显然是f[i-1][j-1]，f[i-1][j-1]+1。但是前两者并不是符合直觉的f[i-1][j],f[i][j-1]：因为这两个集合分别表达“只考虑第一个字符串前i-1个元素和第一个字符串前i个元素”和“只考虑第二个字符串前j-1个元素和第二个字符串前j个元素”，并不是“第一个字符串第i个元素不出现，第二个字符串第j个元素一定出现”的集合。比如f[i-1][j]并不一定包含j。但是可以看到，虽然f[i-1][j]所代表的最大值不一定包含j，这个集合肯定涵盖了所有“只考虑第二个字符串前j-1个元素和第二个字符串前j个元素”，同理可得f[i][j-1]涵盖了对应的陈述。又注意：在这两项涵盖了对应的两个陈述的同时，它们的并集也涵盖了“i，j都不存在”的陈述。所以前三项实际上就是max(f[i-1][j],f[i][j-1])。

char a[N],b[N];
int main()
{
    cin>>n>>m;
    //这样可以直接从角标1开始输入char数组
    scanf("%s%s", a+1, b+1);
    //遍历每一个字符
    for (int i = 1; i <= n; i ++ )
        for (int j = 1; j <= m; j ++ ){
            //得到三种情况中的最大值
            f[i][j]=max(f[i-1][j],f[i][j-1]);
            //两个字符都选的情况
            if(a[i]==b[j]) f[i][j]=max(f[i][j],f[i-1][j-1]+1);
        }
    cout<<f[n][m];
}






5.3 计数类DP

     之前的所有都是求每一集合的最大值最小值，计数类DP求的是个数。

     例题：一个正整数 n 可以表示成若干个正整数之和，形如：n=n1+n2+…+nk，其中 n1≥n2≥…≥nk,k≥1。我们将这样的一种表示称为正整数 n 的一种划分。现在给定一个正整数 n，请你求出 n 共有多少种不同的划分方法。

输入格式
共一行，包含一个整数 n。

输出格式
共一行，包含一个整数，表示总划分数量。由于答案可能很大，输出结果请对 $10^9+7$ 取模。

    这道题可以看作一个完全背包问题，把当前目标值n看作当前背包的容量，共有1～n类物品，价值等于物品编号。集合：f[i][j]，代表当考虑第i个物品的时候，和正好是j的所有加法。属性：由于希望找到所有加法的数量，所以实际上是想找当前集合中元素的数量。计算：集合可以进一步划分为子集：0个该物品/当前值，1个该物品，2个…s个该物品，对应状态方程：f[i][j]=f[i-1][j]+f[i-1][j-i]+f[i-1][j-2i]+...+f[i-1][j-si]。这里探讨的是数量，所以不像背包问题中每一项后面要加上x个当前物品i的价值。

    优化：同样把f[i][j-i]展开： f[i][j-i]=f[i-1][j-i]+f[i-1][j-2i]+...+f[i-1][j-si]（由于是完全背包问题，考量的是背包体积而不是每个物品的个数上限，所以当背包容量和当前物品确定的时候，最后一项一定是一样的，不存在多重背包的问题）。可以看到f[i][j]除了多了第一项，之后的实际上就是后者，所以可以压缩成f[i][j]=f[i-1][j]+f[i][j-i]。最后，同样可以压缩成一维。

const int N = 1010, mod=1e9+7;
int f[N];
int main()
{
    int n;
    cin>>n;
    //当前和为0的加法个数
    f[0]=1;
    for (int i = 1; i <= n; i ++ )   
        //从i开始遍历，防止边界问题
        for (int j = i; j <= n; j ++ )
            f[j]=(f[j]+f[j-i])%mod;
            
    cout<<f[n];
}






5.4 数位统计DP

    给定两个整数 a 和 b，求 a 和 b 之间的所有数字中 0∼9 的出现次数。例如，a=1024，b=1032，则 a 和 b 之间共有 9 个数如下：1024 1025 1026 1027 1028 1029 1030 1031 1032. 其中 0 出现 10 次，1 出现 10 次，2 出现 7 次，3 出现 3 次等等…

输入格式
输入包含多组测试数据。

每组测试数据占一行，包含两个整数 a 和 b。

当读入一行为 0 0 时，表示输入终止，且该行不作处理。

输出格式
每组数据输出一个结果，每个结果占一行。每个结果包含十个用空格隔开的数字，第一个数字表示 0 出现的次数，第二个数字表示 1 出现的次数，以此类推。

    这道题的重点是分类讨论。主要思想是，计算每一个给定数字的每一位上，0～9出现的次数，然后用大的数字减去小的数字的所得的0～9的分别的结果：类似于前缀和计算区间和的思想，比如当a=10，b=8，1在个位上出现的次数等于1-1=0.

    假设有数字abcdefg，现在求千位（d的位置）上1出现的个数，需要分类讨论 xxx1yyy的情况： 1. 前三位数字的范围在000～abc-1时，yyy的取值范围是0～999(不是efg，因为xxx没有取到abc！)，所以总共有abc*1000种 2. 当前三位取到abc时，根据d的不同有三种情况： 1. d < 1 : 0种，因为当d小于想要统计的目标值，显然“在这一位上不可能取到目标值”； 2. d = 1 : efg+1种，因为此时yyy可以取到000～efg； 3. d > 1 : 1000种，因为可以取到abc1000~abc1999。

注：这里的d > 1并不会导致有0种，因为d是原数字本身的值，而“1”指的是目标统计值，当d > 1，证明可以取到1.

//把num的l～r截取出来，用来做乘数
int get(vector<int> num, int l,int r){
    int res=0;
    for (int i = l; i >= r; i -- ){
        res*=10;
        res+=num[i];
    }
    return res;
}

//返回一个10^i
int power10(int i){
    int r=1;
    while(i--){
        r*=10;
    }
    return r;
}
int count(int n, int x){
    if(!n) return 0;
    vector<int> num;
    //为了截取不同的情况作准备
    while(n){
        num.push_back(n%10);
        n/=10;
    }
    //获取数字长度
    n=num.size();
    
    //开始计算个数
    int res=0;
    //-!x是因为当x=0的时候，最高为不可能是0，所以需要直接从次高位开始循环
    for (int i = n-1-!x; i >= 0; i -- ){
        //处理第一种情况：这种情况不可能在最高位发生
        if(i<n-1){
            res+=get(num,n-1,i+1)*power10(i);
            //当x==0，可能会导致0000yyyy真宗情况发生
            //所以需要减去1个power10(i);
            if(!x)res-=power10(i);
        }
        //2.1不存在，不用处理
        //处理第2.2种情况:目标位置与目标数字一样
        if(num[i]==x) res+=get(num,i-1,0)+1;
        //第2.3种情况：
        else if(num[i]>x) res+=power10(i);
    }
    return res;
}

int main()
{
    int a,b;
    
    while(cin>>a>>b,a){
        //前者应当比后者大，为了“区间和”
        if(a > b) swap(a,b);
        
        for (int i = 0; i < 10; i ++ ){
            //a-1是因为目标是a~b的闭区间，所以a也被考虑
            printf("%d ",count(b,i)-count(a-1,i));
        }
        puts("");
    }
}






5.5 状态压缩DP

    蒙德里安的梦想：经典状态压缩DP题目

     求把 N×M 的棋盘分割成若干个 1×2 的长方形，有多少种方案。例如当 N=2，M=4 时，共有 5 种方案。当 N=2，M=3 时，共有 3 种方案。

如下图所示：

输入格式
输入包含多组测试用例。每组测试用例占一行，包含两个整数 N 和 M。当输入用例 N=0，M=0 时，表示输入终止，且该用例无需处理。

输出格式
每个测试用例输出一个结果，每个结果占一行。

数据范围
$1\leq N,M\leq 11$

这道题最重要的思想就是：实际上我们计算的是横向摆放的所有摆法的和，纵向的摆放只是对合法的横向摆放后的填充，所以不用考虑。

从列为出发点查看当前列是否和前一列能构成一个合法的棋盘：每一列都有潜在的 1<< n 种“状态”,比如：

每一种状态都是一个二进制。1 代表着这一列的对应的一行存在横着放（同时向下一列“伸出”一个单位）的矩形，0代表着一行存在竖放/由前一列伸出来的一单位 矩形。当前列是否合法取决于上一列的状态。这里的“上一列”也显然一样有1<<n种状态，比如：

一个合并（对于当前列和前一列的状态的考量）是否合法，需要检查以下两点：

  1. 当前列的状态和前一列状态合并后的列是否有奇数个连续的0。如果有，证明无法把竖的矩形“填充”进去，所以不合法；

  2. 合并后是否有重叠的块

下图前两个是合法的，后两个存在奇数0:

“合并列”就是两个相邻列合并后的列。这两个检查都可以使用位运算实现：

//每一列都有最多2^11种可能的状态，所以M=2<<12
const LL N = 12, M=1<<N;

//st用来保存角标数字代表的二进制是否存在连续奇数个0
bool st[M];
//必须用long long，因为数字太大
LL f[N][M];

int main()
{
    int n,m;
    //可以这样写，输入的同时监测n和m都是0
    while(cin>>n>>m, n||m){
        
        //预处理st数组：
        for (int i = 0; i < 1<<n; i ++ ){
            //记录连续的0的个数
            int cnt=0;
            st[i]=true;
            for (int j = 0; j < n; j ++ ){
                if(i>>j&1){
                    //检验是否奇数
                    if(cnt&1)st[i]=false;
                    //归零计数器
                    cnt=0;
                }
                else cnt++;
        }
        //这里检测的是是否最高项为0
        if(cnt&1) st[i]=false;
    }
    
    //清空前一次问询的结果
    memset(f, 0, sizeof f);
    
    //开始DP部分
    //由于不存在-1列，所以“在第0列不放置横向的矩形”的方法是一个
    //其他第0列的方案都不可能（存在横向的）
    f[0][0]=1;
    //这里需要遍历到m，因为后面取的是f[m][0]
    //目的是遍历每一列
    for (int i = 1; i <= m; i ++ )
        //j和k的循环都是从0到 1<<n-1,是对方案的遍历
        //j：遍历第m列的可能方案
        for (int j = 0; j < 1 << n; j ++ )
            //k：遍历第m-1列的可能方案用来和m列的方案对比是否合法
            for (int k = 0; k < 1 << n; k ++ ){
                //&:假如两个相邻的列的第x位bit都是1
                //在图形上实际上是前一列是横着的，当其列也是横着的
                //会造成1块重叠，所以不合法
                //｜: 同样地，考虑前一列对当前列造成的影响，如果合并之后
                //存在奇数0，不合法；注意这里是st[j|k]而不是对于k/j的单独考量
                if((j&k)==0 && st[j|k]){
                    //累计可行方案数量
                    f[i][j]+=f[i-1][k];
                }
            }
    //输出第m列没有横向矩阵的数量
    cout<<f[m][0]<<endl;
    }
    
}

注：f[0][0]=1的解释：假设给定的图形真的只有一列。显然如果N为偶数的话（合法），有1个合法状态。而这唯一一个可行的方案就是“无横放”。f[0][0]代表的是第0列无横放：显然也便是1了。而只有一列的情况下不可能存在横放，所以其他f[0][x]都是0。初始化第0列的原因：在算法中，可以看到即使真的只有1列，i也会遍历1，同时之前也要给f[0][0]赋值为1（并且默认给其他第0列的状态赋值0）。这是因为后者需要作为前者递推的基础，不然+=前一项的结果永远是0.

    最短Hamilton路径：另一道状态压缩DP：给定一张 n 个点的带权无向图，点从 0∼n−1 标号，求起点 0 到终点 n−1 的最短 Hamilton 路径。Hamilton 路径的定义是从 0 到 n−1 不重不漏地经过每个点恰好一次。

输入格式
    第一行输入整数 n。接下来 n 行每行 n 个整数，其中第 i 行第 j 个整数表示点 i 到 j 的距离（记为 a[i,j]）。
    对于任意的 x,y,z，数据保证 a[x,x]=0，a[x,y]=a[y,x] 并且 a[x,y]+a[y,z]≥a[x,z]。

输出格式
    输出一个整数，表示最短 Hamilton 路径的长度。

    通过输入格式可以看到，给出的是一个完全连通图。状态表示：f[i][j]，表示遍历到节点j，状态为i时的所有汉密尔顿路径；属性：最小值。集合划分：终点为上一个节点k的最短汉密尔顿路径的长度+a[k][j] （所以需要遍历每一个k）。

    这道题和蒙德里安的梦想类似，每一个数字都代表一个状态，比如101可以代表：当前遍历了第一个节点和第三个节点，f[101][2]就代表当遍历了第一和三个节点时的最短汉密尔顿路径。

int w[21][21],f[1<<21][21],n;

int main()
{
    cin>>n;
    for (int i = 0; i < n; i ++ )
        for (int j = 0; j < n; j ++ )
            scanf("%d", &w[i][j]);
            
    memset(f,0x3f,sizeof f);
    //这里代表的是从顶点0到顶点0的距离是0
    //需要初始化是因为如果不：1.在下面的min(f[i][j],f[i-(1<<j)][k]+w[k][j])中，
    //会给它赋INF；2.给后面的数据做递推
    f[1][0]=0;
    //遍历每一种状态
    for (int i = 0; i < 1 << n; i ++ )   
        for (int j = 0; j < n; j ++ ){
            //这里检查当前状态中是否有当前节点，没有的话可以理解为给出的路径/状态/没有覆盖
            //当前节点，所以不会继续运行
            if(i>>j&1){
                //遍历每一个节点，每一个节点都可能是“上一个节点”
                for (int k = 0; k < n; k ++ ){
                    //同上，也要判断是否i中有“上一个节点k”
                    if(i>>k&1) f[i][j]=min(f[i][j],f[i-(1<<j)][k]+w[k][j]);
                }
            }
        }
    //(1<<n)-1得到n个1的二进制数
    cout<<f[(1<<n)-1][n-1];
}






5.6 树形DP

    树形DP，即在树上进行的DP。由于树固有的递归性质，树形DP 一般都是递归进行的。例题：没有上司的舞会：Ural 大学有 N 名职员，编号为 1∼N。他们的关系就像一棵以校长为根的树，父节点就是子节点的直接上司。每个职员有一个快乐指数，用整数 Hi 给出，其中 1≤i≤N。现在要召开一场周年庆宴会，不过，没有职员愿意和直接上司一起参会。在满足这个条件的前提下，主办方希望邀请一部分职员参会，使得所有参会职员的快乐指数总和最大，求这个最大值。

输入格式
第一行一个整数 N。接下来 N 行，第 i 行表示 i 号职员的快乐指数 Hi。接下来 N−1 行，每行输入一对整数 L,K，表示 K 是 L 的直接上司。

输出格式
输出最大的快乐指数。

    DP分析：集合表示f[u][0],f[u][1]：分别表示以u为根节点，u在舞会的happy指数，以及u不在舞会的happy指数；属性：最大值；集合划分：f[u][0]：由于u作为上司不在舞会中，其下属可以在也可以不在舞会，所以取max(f[j][1],f[j][0])，其中j是子节点/下属；f[u][1]：由于u作为上司在舞会中，其下属一定不在舞会，所以直接取f[j][0]。

int happy[N];
bool has_fa[N];

void add(int a, int b){
    e[idx]=a,ne[idx]=h[b],h[b]=idx++;
}
//DP的过程
void dfs(int u){
    //解释为：当以u为根节点，并且该节点被选入（其happy值被纳入统计）
    f[u][1]=happy[u];
    //遍历其子节点
    for(int i=h[u];i != -1;i = ne[i]){
        int j=e[i];
        dfs(j);
        //当没有该节点时，子节点可以在，也可以不在，选最大值
        f[u][0]+=max(f[j][1],f[j][0]);
        //当有该节点是，一定没有子节点
        f[u][1]+=f[j][0];
    }
}

int main()
{
    cin>>n;
    for (int i = 1; i <= n; i ++ ) scanf("%d", &happy[i]);
    memset(h, -1, sizeof h);
    //构建树
    for (int i = 0; i < n-1; i ++ ) {
        int a,b;
        scanf("%d%d", &a, &b);
        add(a, b);
        //最终has_fa[a]!=1的就是老板
        has_fa[a]=1;
    }
    //由于没有告知那个是老板/树根，所以需要求出来
    int root=1;
    while(has_fa[root]) root++;
    
    dfs(root);
    //老板在或不在的最大值
    cout<<max(f[root][1],f[root][0]);
}

在DP过程中的f[u][0]+=max(f[j][1],f[j][0]); f[u][1]+=f[j][0];，虽然看起来假如f[j][0]每次都是最大的话，两个状态的值肯定是一样大的，如果f[j][0]<f[j][1]，那f[u][0]一定大于f[u][1]，所以似乎后者没必要存在。但是由于f[u][1]包含着这个成员本身的happy值，而这个值可能会导致他大于前者。







5.7 记忆化搜索

    例题：滑雪：给定一个 R 行 C 列的矩阵，表示一个矩形网格滑雪场。矩阵中第 i 行第 j 列的点表示滑雪场的第 i 行第 j 列区域的高度。一个人从滑雪场中的某个区域内出发，每次可以向上下左右任意一个方向滑动一个单位距离。当然，一个人能够滑动到某相邻区域的前提是该区域的高度低于自己目前所在区域的高度。下面给出一个矩阵作为例子：

1  2  3  4 5

16 17 18 19 6

15 24 25 20 7

14 23 22 21 8

13 12 11 10 9

在给定矩阵中，一条可行的滑行轨迹为 24−17−2−1。在给定矩阵中，最长的滑行轨迹为 25−24−23−…−3−2−1，沿途共经过 25 个区域。现在给定你一个二维矩阵表示滑雪场各区域的高度，请你找出在该滑雪场中能够完成的最长滑雪轨迹，并输出其长度(可经过最大区域数)。

输入格式
第一行包含两个整数 R 和 C。

接下来 R 行，每行包含 C 个整数，表示完整的二维矩阵。

输出格式
输出一个整数，表示可完成的最长滑雪长度。

集合：f[i][j]代表当前节点为起点的路径的最大值；计算：上下左右的max+1.

int f[N][N],g[N][N],res,n,m;

//偏移量数组
int dx[4]={1,0,-1,0};
int dy[4]={0,1,0,-1};

//DP过程
int dp(int a, int b){
    //通过取地址，把这个槽位当作一个变量v
    int &v=f[a][b];
    //检查是否被遍历过了
    if(v!=-1) return v;
    //本身为1
    v=1;
    //遍历四周，类似DFS
    for (int i = 0; i < 4; i ++ ){
        int x=a+dx[i],y=b+dy[i];
        //检查边界，检查下一个节点是否比当前节点低
        if(x>=1&&x<=n&&y>=1&&y<=m&& g[x][y] > g[a][b]) 
        //递归
        v=max(dp(x,y)+1,v);
    }
    return v;
}
int main()
{
    cin>>n>>m;
    for (int i = 1; i <= n; i ++ )
        for (int j = 1; j <= m; j ++ )
            scanf("%d", &g[i][j]);
    int res=0;
    memset(f,-1,sizeof f);
    //遍历从每一个节点为起始点的路径长度
    for (int i = 1; i <= n; i ++ )  
        for (int j = 1; j <= m; j ++ )
            res=max(res,dp(i,j));
            
    cout<<res;
            
}

这个算法虽然计算了从每一个点为起点的路径长度，但是由于“判断该节点是否被遍历过”导致每个节点实际上都只遍历了一遍。
