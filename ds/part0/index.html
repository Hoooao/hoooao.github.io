导言：

     这篇博客作为Go的学习记录，为完成分布式系统6.824的Lab仅涉及最基础的GoLang知识。6.824曾经使用C++完成Lab，后来切换到了Go。这一举措的原因是Go对“Goroutine”的优秀支持，其RPC包，以及垃圾回收。实际上像是Java之类的语言也可以完成这门课的Labs。







0.1 模组&包&词法分析器 Mod&Package&Lexer

     MOD: 在Go1.11之后，module逐渐代替了workshop的概念，是go的新的dependency/包管理系统。它类似于Node.js中的package.json + package-lock.json。

     在使用IDE创建Go项目之后，使用go mod init <name> 来生成一个go.mod。这个文件定义着当前模组的根目录和当前模组的名字。最开始的go.mod是这样的：

module <name>

go <version>

一个go.mod规定了从这层目录开始的所有目录的module的名字。可以创建子目录，但是要注意 每一个目录都只能从属于一个Package。这里可以看到，N个Package是一个Module的组成部分。如同其他语言，main.go的main()也是入口文件，不需要被显式调用（但是需要从属于package main）。这是一个简单的Go程序：

package main
import "fmt"

func main(){
  fmt.Println("Hello")
}

这个文件没必要在一个子文件夹中，也可以在go.mod同级目录中，但是这样的话其他的xx.go也必须是package main了。

    第一行的package关键字指出这个go文件所属的package。第二行引入了所需的一个标准库fmt。

    下载某个Package类似NPM，比如下载mux：go get -u github.com/gorilla/mux。之后会看到go.mod之下出现了go.sum，实际上就是checksum，用来校验所下载的pkg是否正确。而在go.mod中可以看到一行

require github.com/gorilla/mux v1.8.0 // indirect

保存着pkg的名字和版本，后面的“indirect”的意思是这个pkg并没有被使用。

     Lexer：Go的Lexer很强大，它不光会检测语法，还会处理分号：Go实际上是需要分号于语句的末尾的，但是由于lexer会自动添加，这个分号是不可以手动添加的。







0.2 类型&变量 Types&Variables

    Go中很重要的一个概念就是Type，它可以一定程度上理解为OOP中的Class。Go中有几乎其他语言中都有的那些基本数据类型：

func main() {
	// 创建变量:
	// 显式声明：
	// var + <varName> + <type>

	//String:
	var username string = "test"
	// 模板输出：使用%T作为 type 的占位符
	// Go中函数内部声明的变量必须被使用，不然算作Error！
	fmt.Printf("Var is type: %T\n", username)

	// Bool：
	var logged bool = false

	// Int:
  // 数字相关的类型都可以选择长度
	var smallInt uint8 = 255
	var num int = 1024
	fmt.Println(username, logged, smallInt, num)

	// Float:
	var smallF float32 = 255.66
	var bigF float64 = 2055.66
	fmt.Println(smallF, bigF)

	// Default values and aliases
  // Go中不用担心变量未初始而得到乱码
	var var1 int     // 0
	var var2 string  // ""
	var var3 bool    // false
	var var4 float32 // 0
	fmt.Print(var1, var2, var3, var4)

	// 隐式声明：
	// 不用声明是什么类型，理解为“auto”
	var implicit = "test"
	// implicit = 3 不行：这是赋值，但是没有声明！
	fmt.Print(implicit)

	// 省略 var：达成声明+赋值
	// 使用:=来直接赋值
	number := 3000.9
	fmt.Print(number)

	// 声明const变量，const变量可不被使用且不报错
	const Log = "sss"


	fmt.Print(a, b)
}

    三种变量创建方式：

  • var + <varName> + <type> (= <value>)
  • var + <varName> (= <value>)
  • <varName> := <value>：这一种显然必须赋值。

    同时给多个变量赋：有点类似JS的解构赋值，支持同时使用上述三种变量创建方式。

var a, b int = 1, 22
c, d := 1, "22"

    注： Go中创建的任何变量必须被使用，不然会被Lexer报错：除了const变量。和全局变量。全局变量无法使用:=的方式定义！







0.3 Comma OK || Err

    Go中 不存在Try/Catch，使用Comma OK||Err的模式来“捕捉错误”。这一模式类似于JS中回调函数常常为形如(data,err)=>{}的写法，但不同的是，这里的 OK||Err是返回值。这是一个从console中获取用户输入的简单程序：

func main() {
	fmt.Print("Welcome, type your name:")
	// 这里使用 bufio库来创建一个 Reader，参数是文件描述符，这里是标准输入
	// 这里使用:=赋值，因为不知道返回的类型是什么
	reader := bufio.NewReader(os.Stdin)

	// 这个函数返回两个东西，第一个是输入（String类型 ），第二个是error
	// 当不关心某个返回是什么的时候，使用下划线占位即可
	// ReadString中的参数是deliminator
	input, err := reader.ReadString('\n')

	// 这里是一个很常见的写法，用来检测是否出错
	// nil是Null
	if err!=nil{
		fmt.Print(err)
	}
	else {
		fmt.Print(input)
	}
}

上面的程序中 reader.ReadString('\n') 返回了两个量，使用Go的同时赋值的特性给两个变量赋值。之所以被称之为 Comma OK||Err，是因为通常这种函数的返回值为data, (ok||err)，上面的程序是Comma err的例子。

     **注：**当一个函数返回多个数据的时候，如果有不会使用的变量，必须使用_当作占位符放在目标位置上，不然会因为“为使用该变量”报错。比如上述的例子中我们不关心err，那么就可以写成input, _ := reader.ReadString('\n')，这里的_是一个特殊的保留字，并不会导致报错。
