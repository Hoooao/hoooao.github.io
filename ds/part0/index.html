     这篇博文作为Go的学习记录，为完成分布式系统6.824的Lab仅涉及最基础的GoLang知识。6.824曾经使用C++完成Lab，后来切换到了Go。这一举措的原因是Go对“Goroutine”的优秀支持，其RPC包，以及垃圾回收。实际上像是Java之类的语言也可以完成这门课的Labs。







0.1 模组&包&词法分析器 Mod&Package&Lexer

     MOD: 在Go1.11之后，module逐渐代替了workshop的概念，是go的新的dependency/包管理系统。它类似于Node.js中的package.json + package-lock.json。

     在使用IDE创建Go项目之后，使用go mod init <name> 来生成一个go.mod。这个文件定义着当前模组的根目录和当前模组的名字。最开始的go.mod是这样的：

module <name>

go <version>

一个go.mod规定了从这层目录开始的所有目录的module的名字。可以创建子目录，但是要注意 每一个目录都只能从属于一个Package。这里可以看到，N个Package是一个Module的组成部分。如同其他语言，main.go的main()也是入口文件，不需要被显式调用（但是需要从属于package main）。这是一个简单的Go程序：

package main
import "fmt"

func main(){
  fmt.Println("Hello")
}

这个文件没必要在一个子文件夹中，也可以在go.mod同级目录中，但是这样的话其他的xx.go也必须是package main了。

    第一行的package关键字指出这个go文件所属的package。第二行引入了所需的一个标准库fmt。

    下载某个Package类似NPM，比如下载mux：go get -u github.com/gorilla/mux。之后会看到go.mod之下出现了go.sum，实际上就是checksum，用来校验所下载的pkg是否正确。而在go.mod中可以看到一行

require github.com/gorilla/mux v1.8.0 // indirect

保存着pkg的名字和版本，后面的“indirect”的意思是这个pkg并没有被使用。

     Lexer：Go的Lexer很强大，它不光会检测语法，还会处理分号：Go实际上是需要分号于语句的末尾的，但是由于lexer会自动添加，这个分号是不可以手动添加的。







0.2 类型&变量 Types&Variables

    Go中很重要的一个概念就是Type，它可以一定程度上理解为OOP中的Class。Go中有几乎其他语言中都有的那些基本数据类型：

func main() {
	// 创建变量:
	// 显式声明：
	// var + <varName> + <type>

	//String:
	var username string = "test"
	// 模板输出：使用%T作为 type 的占位符
	// Go中函数内部声明的变量必须被使用，不然算作Error！
	fmt.Printf("Var is type: %T\n", username)

	// Bool：
	var logged bool = false

	// Int:
  // 数字相关的类型都可以选择长度
	var smallInt uint8 = 255
	var num int = 1024
	fmt.Println(username, logged, smallInt, num)

	// Float:
	var smallF float32 = 255.66
	var bigF float64 = 2055.66
	fmt.Println(smallF, bigF)

	// Default values and aliases
  // Go中不用担心变量未初始而得到乱码
	var var1 int     // 0
	var var2 string  // ""
	var var3 bool    // false
	var var4 float32 // 0
	fmt.Print(var1, var2, var3, var4)

	// 隐式声明：
	// 不用声明是什么类型，理解为“auto”
	var implicit = "test"
	// implicit = 3 不行：这是赋值，但是没有声明！
	fmt.Print(implicit)

	// 省略 var：达成声明+赋值
	// 使用:=来直接赋值
	number := 3000.9
	fmt.Print(number)

	// 声明const变量，const变量可不被使用且不报错
	const Log = "sss"


	fmt.Print(a, b)
}

    三种变量创建方式：

  • var + <varName> + <type> (= <value>)
  • var + <varName> (= <value>)
  • <varName> := <value>：这一种显然必须赋值。

    同时给多个变量赋：有点类似JS的解构赋值，支持同时使用上述三种变量创建方式。

var a, b int = 1, 22
c, d := 1, "22"

    注： Go中创建的任何变量必须被使用，不然会被Lexer报错：除了const变量。和全局变量。全局变量无法使用:=的方式定义！







0.3 Comma OK || Err

    Go中 不存在Try/Catch，使用Comma OK||Err的模式来“捕捉错误”。这一模式类似于JS中回调函数常常为形如(data,err)=>{}的写法，但不同的是，这里的 OK||Err是返回值。这是一个从console中获取用户输入的简单程序：

func main() {
	fmt.Print("Welcome, type your name:")
	// 这里使用 bufio库来创建一个 Reader，参数是文件描述符，这里是标准输入
	// 这里使用:=赋值，因为不知道返回的类型是什么
	reader := bufio.NewReader(os.Stdin)

	// 这个函数返回两个东西，第一个是输入（String类型 ），第二个是error
	// 当不关心某个返回是什么的时候，使用下划线占位即可
	// ReadString中的参数是deliminator
	input, err := reader.ReadString('\n')

	// 这里是一个很常见的写法，用来检测是否出错
	// nil是Null
	if err!=nil{
		fmt.Print(err)
	}
	else {
		fmt.Print(input)
	}
}

上面的程序中 reader.ReadString('\n') 返回了两个量，使用Go的同时赋值的特性给两个变量赋值。之所以被称之为 Comma OK||Err，是因为通常这种函数的返回值为data, (ok||err)，上面的程序是Comma err的例子。

     **注：**当一个函数返回多个数据的时候，如果有不会使用的变量，必须使用_当作占位符放在目标位置上，不然会因为“为使用该变量”报错。比如上述的例子中我们不关心err，那么就可以写成input, _ := reader.ReadString('\n')，这里的_是一个特殊的保留字，并不会导致报错。







0.4 型转&时间&打包&内存管理&指针 Conversion&Time&Memo Management&Pointer

     类型转换(Conversion)。在上面的用户输入的例子中，假设用户输入了一个数字，实际上会被解读为字符串传入。但是值得注意的是，由于上面规定以回车为分隔符决定输入的结束，这个回车也会被当作输入传入：

func main() {
	// 这里模拟一个输入，包含着回撤
	input := "12\n"
	convF, err := strconv.ParseFloat(strings.TrimSpace(input), 64)
	// 使用if else检查是否出错
	if err != nil {
		fmt.Print(err)
	} else {
		fmt.Println(convF + 1)
	}
}

这段代码中使用strconv.ParseFloat完成类型转换，使用strings.TrimSpace()删掉了前后的空白（包含回车等字符）。最后的64是结果的bit size。这里也使用了 Comma ok||err的写法。


     Time：是Go中很大的一个包，里面有从Date到sleep各种各样的函数。这里只介绍Date：

func timeMain() {
	// 返回一个Time type
	presentTime := time.Now()
	// Raw:
	fmt.Println(presentTime)
	// Format:
	// 函数内部的参数是格式化的格式：一个递增的字符串01-02-03-04-05-06-07，分别月，日，时，分，秒，年，时区
	fmt.Println(presentTime.Format("01-02-2006 15:04:05 Monday"))
	// 修改这个模板即可改变输出的格式，这里改变年月日为止，且指只输出年的后两位，不输出星期
	fmt.Println(presentTime.Format("06-01-02 15:04:05"))

	// 创建一个时间
	createDate := time.Date(2020, time.January, 30, 23, 59, 59, 59, time.UTC)
	fmt.Println(createDate.Format("06-01-02 15:04:05"))

}

输出：

2022-12-18 20:40:00.344256 -0500 EST m=+0.000137103
12-18-2022 20:40:00 Sunday
22-12-18 20:40:00
20-01-30 23:59:59

Go中的日期格式化很特殊。它是这样规定的：一个递增的字符串01-02-03-04-05-06-07 Monday，分别月，日，时，分，秒，年，时区, 星期。针对不同的输出格式和输出更多的符号（比如“-”），调整数字的前后既可。


     打包（Build）：Go可以很方便地输出MacOS(Darwin), Win, Linux的可执行文件：

首先在终端输入 go env，来查看全局环境变量，可以看到当前的默认OS，这会影响默认的可执行文件的类型；

在终端输入 GOOS = "windows" go build就可以生成一个exe文件，如果不加GOOS = "windows" ，那么就是默认系统。


     内存管理：Go中不需要手动分配内存，释放内存，但是仍然可以手动分配内存。有两个内置函数达到此目的。

    new()：这个函数分配一个未初始化的内存空间，这个空间的内容被0填入，最终返回一个 指针:

type Test struct {
    name string
	yes  bool
}

// 两种定义方式，除了类型不一样，其他没区别
p := new(Test)  // type *Test，name与yes都等于0
var v Test      // type  Test
p.name = "a"
v.name = "b"

new用的很少，其缺点会在后面看到。

    make()：这个函数分配一个初始化的内存空间，这个空间的内容不被0填入，最终返回一个 引用数据类型（不是指针）。它只用在创建slices, maps, and channels这三种情况。

两者对比：

// Unnecessarily complex:
var p *[]int = new([]int)
*p = make([]int, 100, 100)

// Idiomatic:
v := make([]int, 100)

    指针（Pointer）：Go中的指针和C类似：

func ptrMain() {
	// 创建一个int指针，类似C
	// 默认值为nil
	var ptr *int
	num := 23
	// deref
	ptr = &num

	// 修改目标地址的值
	*ptr = *ptr * 20
	fmt.Print(ptr, *ptr, num)
	// 输出 0xc000020100 460 460
}






0.5 数据结构 Data Structures

     Array：Go中几乎不使用Array，因为它几乎没有任何API：

func arrMain() {
	// 声明：var varName [len]Type
	// 必须要提供arr的长度
	// 如此声明的arr并不是ptr！
	var list [20]string
	list[0] = "Apple"
	list[1] = "B"
	list[3] = "X"
	// arr的长度永远是声明的时候的长度，不是保存的数据的长度
	fmt.Println(list, len(list))

	// 声明且赋值
	// 相当于把一个变量赋为arr，而不是给arr赋值
	var list2 = [4]string{"11"}
	fmt.Print(list2)
}

注：假如把一个array作为参数传入，需要形参有一样的长度，类型：

	var list2 = [4]string{"11"}
	// 这是一个匿名函数
	func(arr [4]string) {
		arr[1] = "AAA"
	}(list2)

并且由于array的声明方式，它并不是一个指针，所以原数组值不会变。


     Slice：slice就是Go中的vector，底层通过array实现，有很多API：

func sliceMain() {
	// 两种定义方式：

	// 1. 类似定义并赋值的arr声明，但是没有决定数量
	var slice1 = []string{}
	// 类型为[]string
	fmt.Printf("%T\n", slice1)
	// 添加数据：除了在上面直接赋值，也可以使用append函数添加
	// 但是不能使用[]添加（因为没有分配内存）
	// append的第一个参数是添加的基slice，剩下的是添加的值
	// 可以无限添加，内存自增
	slice1 = append(slice1, "A", "B", "C", "D")
	slice1[2] = "F"
	// 可以Slice 一个 slice：
	// 表达从元素[2]开始slice到最后
	slice1 = append(slice1[2:])
	// 表达从[0]到[1],但是不包含[1]（即exclusive）
	slice1 = append(slice1[:1])
	fmt.Print(slice1)

	// 2. 使用 make() 来创建
	// 注：这里最多只能存4个元素
	slice2 := make([]int, 4)
	// slice2[4] = 12: 报错！
	// 但是使用 append即可：
	// 是基于原slice，append在后面这些数据
	slice2 = append(slice2, 6, 5, 4, 3, 2, 1)
	// 可以使用sort函数，是单独的一个pkg！
	sort.Ints(slice2)
	fmt.Print(sort.IntsAreSorted(slice2))
	// 删除特定index上的数据：删除[2]
	slice2 = append(slice2[:2], slice2[3:]...)

	//遍历Slice：
	// 类似JS中的数组的map
	for v, index := range slice2 {
		fmt.Print(v, index)
	}
	fmt.Print(slice2)
}

加入创建的时候给了初始长度，那么append会在这个长度后面添加。切分slice方式类似python，范围永远是[a,b)。

    第一种方式不能通过index赋值（可能导致segment faults），但是可以在append之后使用index修改数据。第二种方式可以在长度之内使用index赋值，append更多数据可以增加长度。

    注：上面的两种创建slice的方式实际上都只是创建了一个slice struct，这个struct中储存着pointer，指向真正的array。当一个函数的参数是slice的时候，本质上放入了一个slice struct作为实参。修改这个“slice”参数会导致原slice修改：虽然它们是两个struct，但是属性都有指针指向同一个数组。


     Map：也就是哈希表，其声明和Slice一样。

func mapMain() {
	// 定义哈希表，同样是两种方式
	// 1.这里使用类似第二种slice定义的方式
	// []内部的是key，外部的是value
	map1 := make(map[string]int)
	// 2. 这里使用另一种于slice类似的声明
	map2 := map[int]string{1: "1", 2: "2"}

	map1["One"] = 1
	map1["Two"] = 2
	map1["Three"] = 3
	fmt.Println(map1) // Output:map[One:1 Three:3 Two:2]
	// 删除元素：
	delete(map1, "One")

	// Loop through map
	// 类似JS的forEach
	// 同样的，使用_可以占位
	for key, value := range map2 {
		fmt.Println(key, value)
	}
}

输出：

map[One:1 Three:3 Two:2]
1 1
2 2

可以看到哈希表的添加不需要调用函数，直接添加即可。删除指定元素调用delete()即可。对哈希表的遍历类似JS中的forEach。


     结构体（Struct）：Go中的结构体类似C++中的结构体，Go中的结构题实际上可以当作class来看待。Struct的创建：

// User 格式：type strName struct {.. fields}
// struct name 大写
// 由于 其属性是外部可以访问到的，所以也是大写，和const无关
type User struct {
	Name string
	Age  int
	Test func(int) int
}

注：属性大写，是因为在Go中大写的成员表达“public”。

Struct的使用：

func structMain() {
	// 使用struct：
	// 必须要直接定义属性的值，顺序就是定义struct属性的顺序
	// 可以去掉Key，直接写value
	var user = User{Name: "a", Age: 2, Test: func(a int) int { return a }}
	user2 := User{Name: "b", Age: 3, Test: func(i int) int {
		return i
	}}
	fmt.Println(user)
	// 用来输出更细节的信息
	fmt.Printf("%+v", user2)
}

输出：

{a 2 0x108ea20}
{Name:b Age:3 Test:0x108ea40}






0.6 流程控制 Flow Control

     If/Else:

func ifelseMain() {
	num := 22
	// Cond四周没有括号
	// 第一个大括号不能换行
	// 即使只有一个行为也不能像c++一样省略大括号
	if num < 10 {
		fmt.Println("<")
	} else if num > 10 {
		fmt.Println(">")
	} else {
		fmt.Println("=")
	}

	// 可以在if中赋值然后检查这个值
	// 这个值不能被外部使用
	if n := 3; n < 10 {
		fmt.Println(">")
	}
}

总的来说Go中的if/else没任何特殊，但是condition不加括号，并且支持“赋值+condition”这种写法。

     Switch/Case:

func switchMain() {
	n := 10
	switch n {
	// 不需要break，自动
	case 1:
		fmt.Print(1)
	case 10:
		// 但是也可以手动允许执行下面的case
		// 使用 fallthrough关键字
		fmt.Println(10)
		fallthrough
	case 11:
		fmt.Print(11)
		fallthrough
	default:
		fmt.Print("def")
	}
}

**注：**Go与其他语言相反：若一个case不标注fallthrough，那么会自动break。而标注falthrough也只能接着执行下一个case（若后者没有fallthrough），而不是后面的所有case（可以理解为后者隐式调用break）。

     Loop:

func loopMain() {
	arr := []int{1, 2, 3, 4, 5, 6}
	// 第一种：
	// 注：Go不存在++i这种写法
	for i := 0; i < len(arr); i++ {
		fmt.Print(arr[i])
	}
	// 第二种：
	// 这不是 auto i: arr 这种直接遍历整个arr，而只是获得index
	for i := range arr {
		fmt.Print(arr[i])
	}
	// 第三种:
	// 也就是forEach，可以同时获得index和值
	// 如果不关心i，可以用_占位
	for i, n := range arr {
		fmt.Printf("%v %v ", i, n)
	}
	// while loop：
	// 还是使用 for
	i := 0
	for i < 10 {
		// Go也有break，continue
		if i == 4 {
			i++
			continue
		}
		// Go禁止在此处i++！
		fmt.Print(i)
		i++
		if i == 5 {
			break
		}

		// 使用goto
		if i == 3 {
			goto target
		}
	}

	// Goto：
target:
	fmt.Print("Goto")

}

注： Go中不存在while，可以理解为任何while都是for的变体。arrange关键字会返回两个量：index, value，如果只接受一个，那么只是index。Go的goto没什么特殊。







0.7 函数&Defer&文件 Functions&Defer&Files

     函数(Functions)：一个普通函数的定义方式：

// 定义参数
// 返回类型夹在两括号中间
// 返回的时候不会做类型强转，所以类型必须统一
func add(v1 int, v2 int) int {
	// 返回数据
	return v1 + v2
}

     Go中存在类似JS的props的参数传递方式，并且支持多变量返回（Comma OK的基础）：

// 类似扩展运算符
// ...使用在类型的前面
// 可以一次性返回多个！
// 使用小括号包裹所有对应位置上的返回数据的类型
func adds(values ...int) (int, string) {
	res := 0
	for _, v := range values {
		res += v
	}
	return res, " Second Return"
}

这里传入的values将会是一个slice。

    匿名函数：

func (){ fmt.Print(1) }()

匿名函数在其他函数中被使用，会被立即调用。

     方法（Method）：Go中的“方法”实际上知识把一个Type(主要是Struct)当作参数传入的函数，注意区分方法和Receiver。

// Method:Type为"Receiver"，不是OOP中“Class/Obj的函数”
// 这里传入的User将会是一份copy，而不是真正的user对象
func (u User)getName() string {
	// 原struct的Name不会被改
	u.Name = "B"
	return u.Name
}
func methodMain() {
	user := User{"A", 11}
	// 调用方式和OOP中的方法一样
	fmt.Print(user.getName())
}

     Defer：Defer是Go中很重要的关键字，它会让后面的语句在当前函数结束/返回前执行：

// Defer
func deferMain() {
	// Deferred 1先输出
	defer fmt.Println("Deferred 2")
	fmt.Println("1")
	defer fmt.Println("Deferred 1")

	fmt.Println(deferFunc())

	fmt.Println("2")
}

func deferFunc() string {
	for i := 0; i < 5; i++ {
		defer fmt.Print(i)
	}
	return "na"
}

在执行的过程中，defer的语句会被放入一个“Stack”，在函数结束之前LIFO执行，所以上述例子中Deferred 1先执行。deferMain()中调用了deferFunc()，后者中有defer，但是由于它本身不是defer，所以deferFunc()本身不会等到其外部函数结束前执行：但它内部的defer仍会按照对应的规则在它结束之前执行。

输出：

1
43210na
2
Deferred 1
Deferred 2

na后于43210输出就是return后于defer执行的体现。

     Files：Go中的文件读写主要依赖于两个包：io,os：

func fileWriteMain() {
	content := "File Contents"
	// 生成一个fd，或报错
	file, err := os.Create("./myText.txt")

	// 通常关闭资源的语句使用defer
	defer file.Close()
	
	if err != nil {
		// Shut down process, show err
		panic(err)
	}

	length, err := io.WriteString(file, content)
	fmt.Print(length)
	fileReadMain()
}

func fileReadMain() {
	// 读取的内容并不是string，而是bytes
	dataBytes, err := os.ReadFile("./mytext.txt")
	if err != nil {
		panic(err)
	}
	// 有两种转换成steing的方式：其一：
	fmt.Print("output:", string(dataBytes))
}

文件写入需要创建FD，写入返回的是长度，使用Comma OK模式得到报错。读入文件不需要FD，返回的内容是二进制bytes，不是编码的制服穿，使用string()函数型转即可。

注： 关闭资源使用defer，防止提前关闭或者忘写关闭。Go的垃圾回收会清理打开的文件，但是这些文件还是会在程序运行期间浪费资源，如果程序过大，影响也会可见。







0.8 并发&互斥锁&管道 Concurrency&Mutex&Channel

     并发 是Go中最关键的特性：

func main() {
	// 第一个例子：输出数字
	// 加关键字go创建一个goroutine
	// 这个线程独立执行，但是main会在执行完第二个sequential的goMain之后退出
	// 所以如果不sleep就看不到goroutine的输出
	go goMain()
	goMain()
}

func goMain() {
	for i := 0; i < 10; i++ {
		// 这里sleep就可以看到另一个goroutine的输出了
		time.Sleep(10 * time.Millisecond)
		fmt.Print(i, " ")
	}
}

在上述例子之中，sleep是必要的，假如没有这个延时，会导致看不到开启的GoRoutine的输出，这是因为 只要主线程（Caller）结束，那么分线程会立马结束：这是很重要的一个特性！而Sleep使得分线程有机会执行，但是睡眠时间太短，仍然可能造成直接关闭，而太长会浪费时间。解决方法：WaitGroup。

第二个例子：

// 使用sync.WaitGroup
var wg sync.WaitGroup

func main() {

	// 第二个例子：多线程Fetch网页
	wl := []string{
		"https://www.bilibili.com/",
		"https://www.youtube.com/",
		"https://www.zhihu.com/",
		"https://www.google.com"}
	for _, v := range wl {
		// 如果直接go getRequest()不会出现输出
		// 需要把goroutine加入wait group中等待！
		go getRequest(v)

		// 这里counter++
		wg.Add(1)
	}
	// 当counter == 0， 停止等待
	// 相比goMain中的sleep更加合理
	wg.Wait()
}

// 异步函数
func getRequest(site string) {
	// 这里使用Done让counter--
	// defer放在开头，类似file.Close()的原因
	defer wg.Done()
	_, err := http.Get(site)
	if err != nil {
		fmt.Print(err)
	} else {
		fmt.Println("202： ", site)
	}
}

在这个例子之中使用了sync.WaitGroup，它本质上是一个Struct，维护着一个counter，是对“正在运行的GoRoutine”的纪录：一个线程开始则+1（wg.Add(1)），结束则-1 (wg.Done())。Add函数可以同时增加这个counter任意数字，而Done只是-1。主进程最后的wg.Wait()类似于JS中的Promise.All()，等待所有promise函数结束：这里的Wait等待自己的counter减少为0（负数会报错，Don!=Add(n)会导致永远Wait）。

     互斥锁：

func main() {
	// 无论锁还是WG实际上通常在使用的时候都是通过指针传递的
	// Go中无论是指针还是值，都可以直接使用.来访问属性，所以在调用上没什么区别
	// 注意末尾的{}，这是在创建Struct实例
	mut := &sync.Mutex{}
	wg := &sync.WaitGroup{}
	raceMain(mut, wg)
}

// 创造一个lock
var a int = 0

// 标注型参是指针
func raceMain(mut *sync.Mutex, wg *sync.WaitGroup) {
	for i := 0; i < 1000000; i++ {
		wg.Add(1)
		go addA(mut, wg)
	}
	wg.Wait()
	fmt.Println(a)
}

func addA(mut *sync.Mutex, wg *sync.WaitGroup) {
	defer wg.Done()
	// 需要标出Critical Region
	mut.Lock()
	a++
	mut.Unlock()
}

锁的两个部分就是上锁和解锁，把这两个函数套在Critical Region外部即可。在上述的例子中，很重要的一点就是，mutex和wg都是通过指针传递的，这是必要的。因为假设wg是值传递，那么wg.Wait()会直接导致程序永远Wait：Done并不作用于这个最开始的wg。而对mutex来说实际上每个addA都有自己的锁，并无法保护a。

     管道（Channel）：管道就是操作系统中的Pipe，实现线程之间的通信（不是进程通信）。管道的创建和Slice，Map一样，使用make创建。这里是一个 错误示例 ：

// 错误示例
func wrongMain() {
	// 创建Channel
	ch := make(chan int)
	// 这样会造成死锁
	func() { ch <- 5 }()
	fmt.Print(<-ch)
}

上面的代码执行匿名函数，这个匿名函数把一个数字放入了channel。这样会导致死锁。首先，这里使用make(chan int)创建了一个管道，这个管道实际上是没有Buffer的，也就是说，数据被写入之后会等待输出。但是直到下一行管道才会被读取输出。而当一个管道有了输入没有输出的时候，会进入睡眠（输入方被阻塞），在这个例子中，“所有GoRoutine”都进入了睡眠，也就是这个语境下的“死锁”。

func chanMain() {
	// 创建Channel：
	// 使用make来创建，第二个参数是channel的buffer的大小
	ch := make(chan int, 2)
	// Channel是异步的，所以需要使用wg
	wg := &sync.WaitGroup{}
	wg.Add(2)
	// 创建监听的func：
	go func(ch chan int, wg *sync.WaitGroup) {
		time.Sleep(3 * time.Second)
		fmt.Print(<-ch, <-ch)
		// ch <- 9
		wg.Done()
	}(ch, wg)

	go func(ch chan int, wg *sync.WaitGroup) {
		ch <- 2
		ch <- 10
		// 这里不能读入，因为只有两个输入
		// fmt.Print(<-ch)
		wg.Done()
	}(ch, wg)
	wg.Wait()
}

在上面的例子中，给了管道 Capacity/Buffer，为2。之后创建了2个GoRoutine分别去执行接收和发送。

注： 编译器会检测到“这里存在数量对等的接收方和发送方”，因此不会报错（不要设想可能存在其中一个GoRoutine慢了一步，让另一个GoRoutine以为又是死锁）：假如之前错误的例子中的匿名函数也是一个单独的Goroutine，那么可正常运行。通常来说不会使用buffer为1的chan，可能会有错误。

func closeMain() {
	ch := make(chan int, 2)
	wg := &sync.WaitGroup{}
	wg.Add(2)

	go func(ch <-chan int, wg *sync.WaitGroup) {
		// 由于close发出了0，这里可以得到那个0
		// 输出：2 10 0
		// fmt.Print(<-ch, <-ch, <-ch)

		// 可能出现chan<-0，但是也可能是close导致的
		// 区分方法：
		for {
			// 实际上<-ch会返回两个变量：数据和是否open
			val, isOpen := <-ch
			if !isOpen {
				break
			}
			fmt.Println(val)
		}
		wg.Done()
	}(ch, wg)

	go func(ch chan<- int, wg *sync.WaitGroup) {
		ch <- 2
		ch <- 10
		// close关闭整个channel，只能由发送方调用！
		// close 会chan<-0，即使超出buffer长度
		close(ch)
		wg.Done()
	}(ch, wg)
	wg.Wait()
}

在管道被关闭之后，读取方可以得到一个0.此时需要辨别到底是输入方输入了一个0还是close(ch)被调用了：实际上对于管道的读取会返回两个值，一个是buffer中的下一个值，另一个是一个值表达“管道是否被关闭”。这个最后的0不算在管道的长度之中，不会导致超出上限。需要质疑的是，在管道关闭之后，还可以读，但是不能写。

     两个常用API：len(ch)返回管道当前缓存的值的数量; cap(ch)返回初始化时所填入的管道的容积。







0.9 OOP in GO

    Go 并不是一个严格意义上的OOP，但是它实现了很多类似普通OOP也会有的概念，比如继承，多态，方法等。


     继承/组合（Inheritance/Composition）：Go中实际上并不存在继承，但是实现了“组合”，于继承很类似：

// Parent Go中的继承不使用类似extends的关键字，而是对某个Struct的匿名结构体使用
// 这里定义了一个"父类"
type Parent struct {
	Name string
	age  int
}

// 这里定义一个Parent的方法
func (p Parent) parentMethod() int {
	return p.age
}

// Child 这里定义了一个"子类"
type Child struct {
	// 匿名结构体，实现对于"父类"属性继承
	Parent
	Num  int
	Addr string
}

func inheriMain() {
	// 先创建一个空的child对象
	// 不能一层大括号直接赋值，匿名结构体需要显式声明！
	// Parent的age是私有属性，但是仍然可以访问！
	child := Child{Parent{"a", 11}, 1, "X"}

	// 子类可以调用父类的方法
	child.Parent.parentMethod()

	// 另一种赋值方式
	child2 := Child{}
	child2.Addr = "X"
	child2.Name = "a"
	child2.Num = 2
	child2.age = 11
	child2.parentMethod()
}

所以本质上可以把Go的组合理解为一个struct中的成员变量是一个匿名struct，然后通过这个匿名struct调用目标struct的方法/变量等等，类似于Less中直接把一个样式对象直接拷贝到另一个样式里。虽然OOP中私有变量无非被继承，这里可以。


     方法/Receiver：Go中的Struct中的函数无法访问到自己的成员变量！需要使用 Receiver/方法来达成这一目的：

// GetName 定义一个方法
// 在func后面插入 ( <VarName> <Structname>)，就可以通过VarName访问"当前"struct对象了
// 类似手动标志了"谁是this"和"谁能调用这个函数"
func (u User) GetName() string {
	return u.Name
}

// 这个Receievr使用的是指针！
func (u *User) SetName() {
	u.Name = "B"
}
func receiverMain() {
	user := User{Name: "b", Age: 1, Test: func(a int) int { return a }}
	// 上面Receiver第一个括号不需要显式表达
	user.SetName()
	fmt.Print(user.GetName())
}

如果想要修改Struct中的值，需要使用指针！

注： Go中任意Type都可以拥有方法，函数前面的括号实际上在指出“之后这个类型的数据可以调用这个函数”。


     Interface：Go中的struct不存在泛型，而interface一定程度上弥补了这一空缺。一个Interface定义了一个函数集合，任何有同样函数作为Receiver的Struct，都会被这个Interface变成同一个泛型：

// Animal 定义泛型：
// type <interface name> interface {...required funcs}
type Animal interface {
	GetName() string
}

这里定义了一个叫做Animal的Interface，相当于声明任何有GetName() string作为Receievr的Struct，都可以被看作Animal处理。

// 现在有两个struct
type Dog struct {
	Name string
}
type Cat struct {
	Name string
}

// GetName 它们有一样的函数
func (d Dog) GetName() string {
	return d.Name
}
func (c Cat) GetName() string {
	return c.Name
}

这里定义了两个Struct，它们有相同的GetName() string，因此它们都是Animal，可以在这个函数中使用：

// 它们会被当作一种type(interface name)看待！

func getInfo(a Animal) {
	fmt.Print(a)
}

func interfaceMain() {
	c := Cat{"caat"}
	getInfo(c)
}

    在Go的源码中的使用：func Sort(data Interface)，sort函数的型参就是一个interface，包含函数Len，Less，Swap。

注：一个Struct可有更多的Receiver，但必须拥有目标Interface中的所有Receiver才能被看作这一Interface定义的泛型。另外，如果一个Interface是一个空集（内部没有任何函数），那么任何Type都是这个Interface的泛型！这种接口的实现体现了Go的 高内聚，低耦合。

    另一个接口的例子：

// Device 定义Interface：
// 满足该interface定义的方法集合的struct被归类为同一个interface
type Device interface {
	TurnOn()
}

type Phone struct {
	Name string
}

func (p Phone) TurnOn() {
	fmt.Println(p.Name, "开机")
}

func (p Phone) Shot() {
	fmt.Println(p.Name, "关机")
}

type Camera struct {
	Serial int
}

func (c Camera) TurnOn() {
	fmt.Println("开机", c.Serial)
}

func (c Camera) TurnOff() {
	fmt.Println("关机", c.Serial)
}
func deviceFunc(d Device) {
	fmt.Println(d)
}

func interfaceMain() {
	p := Phone{"Phone"}
	c := Camera{11}
	deviceFunc(p)
	deviceFunc(c)
}

接口不可以单独产生一个“该接口类型的变量”，但是可以实现指向一个符合接口规范的实例:

var d Device = Phone{"D"}
deviceFunc(d)

     类型断言：也就是向下型转：把泛型转换成某个特定的自定义类型。常常在函数返回了一个空接口（可以是任意类型）后使用：

// 类型断言
func assertionMain() {
	p := Phone{"Phone"}
	// 这样可以直接转换：向上型转
	var d Device = p
	// 这样不可以：向下型转
	// var p2 Phone = d
	// 使用类型断言，类似其他语言中的(type)Var
	var p2 Phone = d.(Phone)

	// 也存在Comma OK，强转失败报错
	p3, err := d.(Phone)
	fmt.Print(p2, p3, err)
}






0.9 远程过程调用 RPC: Remote procedure call

     RPC是一种通信协议，允许一台计算机的程序调用另一个地址空间的子程序/函数，Go中自带rpc包。在这种协议下，存在服务器和用户，服务器可以选择 注册（Register） 本程序中的Object的方法，在注册之后，这些方法可以被用户们使用。一个方法必须满足以下几个需求才能被使用：

the method’s type is exported.

the method is exported.

the method has two arguments, both exported (or builtin) types.

the method’s second argument is a pointer.

the method has return type error.

综上，一个正常的可被使用的函数形如：func (t *T) MethodName(argType T1, replyType *T2) error。其中第一个参数是Caller提供的参数，第二个是这个函数返回给Caller的类型：返回值通过参数（指针）传递，也就是RPC中“交流”的关键。例子：

// 服务器：创建Registration暴露方法
package server

import "errors"

// Caller提供的参数类型
type Args struct {
	A, B int
}

// Caller将会接受的返回类型
type Quotient struct {
	Quo, Rem int
}

type Arith int

func (t *Arith) Multiply(args *Args, reply *int) error {
	*reply = args.A * args.B
	return nil
}

func (t *Arith) Divide(args *Args, quo *Quotient) error {
	if args.B == 0 {
		return errors.New("divide by zero")
	}
	quo.Quo = args.A / args.B
	quo.Rem = args.A % args.B
	return nil
}

arith := new(Arith)
// 注册Object：Arith的两个方法被publish
rpc.Register(arith)
rpc.HandleHTTP()
// 得到Listener：创建了服务器
l, e := net.Listen("tcp", ":1234")
if e != nil {
	log.Fatal("listen error:", e)
}
// 接收HTTP连接请求，给每个连接分配一个goroutine
go http.Serve(l, nil)


// 用户：发起对服务器的链接
client, err := rpc.DialHTTP("tcp", serverAddress + ":1234")
if err != nil {
	log.Fatal("dialing:", err)
}
// 这里是pkg调用对方的Struct，不是RPC
args := &server.Args{7,8}
var reply int
// 调用Call函数，使用目标函数
err = client.Call("Arith.Multiply", args, &reply)
if err != nil {
	log.Fatal("arith error:", err)
}
fmt.Printf("Arith: %d*%d=%d", args.A, args.B, reply)






0.10 Go内存模型 Go Memory Model

     这门课中的“多线程”并不是为了性能，而是为了实现分布式的思想，所以不要关注于“怎样提高多线程的CPU性能”，正确比高性能要重要。这部分不解释Go的内存模型，更多的是机遇内存模型的代码技巧

0.10.1 闭包 Closure

     Go中的闭包和JS是类似的。当一个函数调用外部的变量的时候，它会把这个变量考量到自己的生命周期里，即使外部的函数结束，但是这个内部函数没有，那么这个“外部变量”也不会消失:

func main() {
	wg := sync.WaitGroup{}
	for i := 0; i < 5; i++ {
		wg.Add(1)
		go func(x int) {
			innerFunc(x)
			wg.Done()
		}(i)
	}
	wg.Wait()
}

func innerFunc(x int) {
	fmt.Println(x)
}

输出：

0
4
1
2
3

这是一个正确的例子，匿名函数在被调用的时候会隐式调用 x = i，x不会跟着外部的变量i变化。但是下面这个类似的例子会导致大概率输出4或5个5:

func main() {
	wg := sync.WaitGroup{}
	for i := 0; i < 5; i++ {
		wg.Add(1)
		go func() {
			innerFunc(i)
			wg.Done()
		}()
	}
	wg.Wait()
}

func innerFunc(x int) {
	fmt.Println(x)
}

这是因为 x = i是在innerFunc这个goroutine内部执行的，当这一步被执行的时候，i以及被遍历到了4或者5。

0.10.2 Condition variable

     很常见的一个情景是，希望在某个条件被满足之后，通知一些进程去做有些事情，在此之前这些进程应当等待这个条件满足：

// 定义全局变量
var done bool
var mu sync.Mutex

func main() {
	go periodic()
	time.Sleep(5 * time.Second) // wait for a while so we can observe what ticker does
	mu.Lock()
	// 修改done的值，激活periodic
	done = true
	mu.Unlock()
	println("cancelled")
	time.Sleep(3 * time.Second) // observe no output
}

// 这函数等待done变成true之后返回
func periodic() {
	// polling
	for {
		println("tick")
		// sleep
		time.Sleep(1 * time.Second)
		// 需要使用lock来避免 read-write race
		mu.Lock()
		if done {
			return
		}
		mu.Unlock()
	}
}

另一个更复杂的例子是统计投票：

func main() {
	rand.Seed(time.Now().UnixNano())

	count := 0
	finished := 0
	var mu sync.Mutex

	for i := 0; i < 10; i++ {
		// 这里发10个Goroutine去随机投票
		go func() {
			vote := requestVote()
			mu.Lock()
			defer mu.Unlock()
			if vote {
				count++
			}
			finished++
		}()
	}

	for {
		mu.Lock()
		// 当赞成个数超过一半，就没必要再等了
		if count >= 5 || finished == 10 {
			break
		}
		mu.Unlock()
		// 这里是为了防止busy-wait
		time.Sleep(50 * time.Millisecond)
	}
	if count >= 5 {
		println("received 5+ votes!")
	} else {
		println("lost")
	}
	mu.Unlock()
}

func requestVote() bool {
	time.Sleep(time.Duration(rand.Intn(100)) * time.Millisecond)
	return rand.Int() % 2 == 0
}

     这样的代码有一个缺陷：为了避免busy-wait采用了sleep，而sleep的长度是一个任意决定的数字，可能太大/太小而浪费性能。Go中提供了一种避免这种随机设置等待市场的方式：Condition variable：

func main() {
	rand.Seed(time.Now().UnixNano())

	count := 0
	finished := 0
	var mu sync.Mutex
	// 这里创建了一个Condition Variable
	// 一个Cond需要一个mutex作为自己的属性
	cond := sync.NewCond(&mu)

	for i := 0; i < 10; i++ {
		go func() {
			vote := requestVote()
			mu.Lock()
			if vote {
				count++
			}
			finished++
			// 这里cond会把等待这个cond的所有goroutin唤醒
			cond.Broadcast()
			mu.Unlock()
		}()
	}

	mu.Lock()
	for count < 5 && finished != 10 {
		// 这里这个goroutine会放弃mu锁，进入睡眠
		cond.Wait()
	}
	if count >= 5 {
		println("received 5+ votes!")
	} else {
		println("lost")
	}
	// 在唤醒之后会重新获得lock，这里释放
	mu.Unlock()
}

func requestVote() bool {
	time.Sleep(time.Duration(rand.Intn(100)) * time.Millisecond)
	return rand.Int() % 2 == 0
}

     一个cond需要一个mutex指针作为自己的属性，可以理解为：当cond被使用的时候，首先释放这个mutex，当cond为真的时候，唤醒等待着的goroutine，并且把mutex还给它们。cond.Broadcast()必须在critical section中被调用，不然可能造成Lost wakeup。另外，存在**cond.Signal()**来唤醒1个正在等待的Goroutine，一般不用。

0.10.3 Channel

     虽然逻辑上chan和queue很像，但是chan在实践中是 Synchronous 的，也就是说，它不需要显式地使用Mutex来保护其输入输出：

package main

import "time"
import "fmt"

func main() {
	c := make(chan bool)
	go func() {
		// 这会block外部的main groutine
		time.Sleep(1 * time.Second)
		<-c
	}()
	start := time.Now()
	c <- true // blocks until other goroutine receives
	fmt.Printf("send took %v\n", time.Since(start))
}

这是一个 无Buffer 的chan。直觉上，似乎在执行完c <- true之后程序就会输出Printf并结束，但实际上它会等待内部的goroutine执行完毕（sleep 1s）才会执行Printf输出并结束：chan知道当前程序中是否有配对的输出和输入，如果有的话，在输出之后才会继续执行，不然就block输入的那一方。错误例子：

func main() {
	c := make(chan bool)
	c <- true
	<-c
}

在这个例子中，虽然有一对输入输出，但是在输入结束之后当前goroutine会进入block，导致下一行的输出永远无法执行：死锁。

注： 在一对输入输出中，等待是相互的：输入先出现会等待输出，反之亦然。

     可以利用chan的输入输出相互等待的特性来实现WaitGroup：

func main() {
	done := make(chan bool)
	for i := 0; i < 5; i++ {
		// 这里生成了5个希望等待的goroutine
		go func(x int) {
			sendRPC(x)
			// 这里最终会输入5个元素
			done <- true
		}(i)
	}
	// 这里存在相同数量的读入
	// 在不存在数据的时候会进入的等待
	for i := 0; i < 5; i++ {
		<-done
	}
}
func sendRPC(i int) {
	println(i)
}

上面的代码中，也可以使用长度为5的 buffer channel，但是不推荐使用buffer channel。实际上channel本身都不推荐使用。
